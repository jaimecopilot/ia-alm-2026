# P4.6 – DETECCIÓN Y CORRECCIÓN DE BUGS CON IA

Actúa como un diseñador instruccional experto en formación de desarrolladores de software. Necesito que generes un ejercicio práctico detallado para un curso de IA aplicada al desarrollo de software.

## Contexto del módulo

El módulo es "IA en Desarrollo y Codificación". Los alumnos están aprendiendo a usar IAs para detectar y corregir bugs en el código, simulando el flujo de auto-reparación que ocurre en el ALM generativo.

## Concepto clave a transmitir

En el ALM generativo, los agentes de IA no solo generan código: también ejecutan tests, detectan fallos y los corrigen automáticamente (bucle de auto-reparación). El desarrollador debe entender este proceso para supervisarlo eficazmente. La IA puede actuar como un debugger inteligente que analiza el código, identifica la causa raíz y sugiere una corrección.

## Objetivo del ejercicio

Que los alumnos aprendan a usar IAs para detectar bugs en código (tanto bugs evidentes como sutiles), diagnosticar la causa raíz y generar correcciones verificables mediante tests.

## Escenario

Proporciona varios fragmentos de código con bugs de diferentes tipos: error off-by-one, condición invertida, race condition simulada, tipo de dato incorrecto, y un bug lógico sutil que no produce error de ejecución pero da resultados incorrectos.

## Estructura que debes generar

### 1. Introducción

Explica el concepto de bucle de auto-reparación en el ALM generativo: generar → testear → fallar → diagnosticar → corregir → repetir. El desarrollador supervisa este ciclo.

### 2. Prompt 1: Detección de bug evidente

Proporciona un fragmento de código con un bug evidente (error off-by-one o condición invertida) y un prompt para que la IA lo detecte, explique la causa y proponga una corrección.

### 3. Prompt 2: Detección de bug sutil

Proporciona un fragmento con un bug lógico que no produce error pero da resultados incorrectos (ej: fórmula matemática errónea, filtro con condición mal escrita). Pide a la IA que lo detecte y corrija.

### 4. Prompt 3: Diagnóstico de bug con stack trace

Proporciona un mensaje de error (stack trace) junto con el código fuente. Pide a la IA que diagnostique la causa raíz a partir del error y proponga la corrección.

### 5. Prompt 4: Corrección con tests de regresión

Proporciona un prompt para que la IA, después de corregir el bug, genere tests que verifiquen que el bug está corregido y que no se introducen regresiones.

### 6. Tabla de evaluación

Crea una tabla con columnas para cada tipo de bug y filas para: ¿la IA detectó el bug?, ¿diagnosticó correctamente la causa?, ¿la corrección era correcta?, ¿los tests generados verifican la corrección?

### 7. Reflexión final

Pregunta al alumno: ¿qué tipo de bugs fueron más difíciles de detectar para la IA?, ¿corregiría la IA bugs sin supervisión humana?, ¿cómo se relaciona este ejercicio con el bucle de auto-reparación del ALM generativo?

## Reglas de formato

NO uses la palabra "minuto". Estructura el ejercicio en "Paso 1", "Paso 2", etc.

El ejercicio debe ocupar al alumno aproximadamente 15 minutos.

Los prompts deben estar en bloques de código Markdown.

Incluye los fragmentos de código con bugs y espacios para las correcciones.

Los prompts deben ser genéricos y funcionar en cualquier IA.

## Tono

Profesional pero cercano. Enfatiza que la detección de bugs es una responsabilidad compartida entre la IA y el desarrollador.
