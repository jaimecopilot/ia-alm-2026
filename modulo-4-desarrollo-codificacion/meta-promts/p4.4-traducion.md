# P4.4 – TRADUCCIÓN DE CÓDIGO ENTRE LENGUAJES

Actúa como un diseñador instruccional experto en formación de desarrolladores de software. Necesito que generes un ejercicio práctico detallado para un curso de IA aplicada al desarrollo de software.

## Contexto del módulo

El módulo es "IA en Desarrollo y Codificación". Los alumnos están aprendiendo a traducir código entre lenguajes de programación usando IAs, manteniendo la lógica y adaptando las convenciones idiomáticas.

## Concepto clave a transmitir

La traducción entre lenguajes con IA va más allá de la conversión sintáctica. Implica adaptar las convenciones idiomáticas (camelCase a snake_case, getters/setters a propiedades, bucles a list comprehensions), convertir estructuras de datos al equivalente nativo y ajustar el manejo de errores. La IA debe actuar como un traductor cultural, no solo sintáctico.

## Objetivo del ejercicio

Que los alumnos aprendan a traducir código entre lenguajes (Java a Python, Python a JavaScript) manteniendo la lógica original y adaptando las convenciones idiomáticas, y a verificar la equivalencia funcional mediante tests.

## Escenario

Proporciona un fragmento de código en Java (una clase con lógica de negocio) y pide traducirlo a Python. Luego proporciona un fragmento en Python y pide traducirlo a JavaScript.

## Estructura que debes generar

### 1. Introducción

Explica la diferencia entre traducción sintáctica y traducción idiomática. La IA debe adaptar el código al estilo del lenguaje destino.

### 2. Prompt 1: Traducción Java a Python

Proporciona el código Java y un prompt para traducirlo a Python especificando las adaptaciones idiomáticas: getters/setters a acceso directo, camelCase a snake_case, excepciones checked a runtime, añadir type hints y docstrings.

### 3. Prompt 2: Verificación de equivalencia

Proporciona un prompt para generar tests equivalentes en ambos lenguajes y verificar que producen los mismos resultados.

### 4. Prompt 3: Traducción Python a JavaScript

Proporciona un fragmento Python y un prompt para traducirlo a JavaScript moderno, especificando las adaptaciones: snake_case a camelCase, type hints a JSDoc, tuplas a objetos.

### 5. Tabla comparativa

Crea una tabla con columnas para el lenguaje original y el traducido, y filas para: legibilidad, fidelidad de la lógica, uso de convenciones idiomáticas, manejo de errores.

### 6. Reflexión final

Pregunta: ¿qué desafíos encontraste en la traducción?, ¿la IA respetó las convenciones idiomáticas del lenguaje destino?, ¿cómo verificarías que el código traducido es correcto?

## Reglas de formato

NO uses la palabra "minuto". Estructura el ejercicio en "Paso 1", "Paso 2", etc.

El ejercicio debe ocupar al alumno aproximadamente 15 minutos.

Los prompts deben estar en bloques de código Markdown.

Incluye el código original y espacios para el código traducido.

Los prompts deben ser genéricos y funcionar en cualquier IA.

## Tono

Profesional pero cercano.
