# P4.7 – GENERACIÓN DE CÓDIGO APLICANDO PATRONES DE DISEÑO

Actúa como un diseñador instruccional experto en formación de desarrolladores de software. Necesito que generes un ejercicio práctico detallado para un curso de IA aplicada al desarrollo de software.

## Contexto del módulo

El módulo es "IA en Desarrollo y Codificación". Los alumnos están aprendiendo a usar IAs para generar código que implementa patrones de diseño, acelerando la creación de arquitecturas bien estructuradas.

## Concepto clave a transmitir

Los patrones de diseño son soluciones probadas a problemas recurrentes. La IA conoce la estructura canónica de cada patrón y puede implementarlo en segundos si se le especifica correctamente. El nombre del patrón actúa como un atajo semántico que carga todo un esquema de diseño en el contexto del modelo. Esto es especialmente potente en el ALM generativo: puedes pedir a un agente que implemente una feature siguiendo un patrón concreto, y el agente generará el código con la estructura correcta.

## Objetivo del ejercicio

Que los alumnos aprendan a pedir a la IA que genere código aplicando patrones de diseño específicos (Repository, Service, Observer, Strategy, Factory) y a evaluar si la implementación generada sigue correctamente la estructura del patrón.

## Escenario

Proporciona un contexto de sistema de reservas donde se necesitan implementar varios patrones: Repository para acceso a datos, Service para lógica de negocio, Observer para notificaciones, Strategy para cálculo de precios, y Factory para creación de objetos.

## Estructura que debes generar

### 1. Introducción

Explica que los patrones de diseño son un lenguaje común entre desarrolladores e IAs. Nombrar un patrón en el prompt dispara una implementación estructurada.

### 2. Prompt 1: Patrón Repository

Proporciona un prompt para que la IA genere una implementación del patrón Repository para una entidad (ej: Sala), con métodos CRUD estándar y consultas personalizadas.

### 3. Prompt 2: Patrón Service + Inyección de dependencias

Proporciona un prompt para generar un Service que use el Repository generado, con inyección de dependencias y lógica de negocio.

### 4. Prompt 3: Patrón Observer para notificaciones

Proporciona un prompt para implementar el patrón Observer donde el servicio de reservas notifica a suscriptores (EmailNotifier, PushNotifier) cuando se crea una reserva.

### 5. Prompt 4: Patrón Strategy para cálculo de precios

Proporciona un prompt para implementar el patrón Strategy con diferentes algoritmos de cálculo de precios (tarifa estándar, premium, por horas).

### 6. Tabla de evaluación

Crea una tabla con columnas para cada patrón implementado y filas para: ¿la estructura del patrón es correcta?, ¿los nombres de clases/métodos son los canónicos?, ¿el código es funcional?, ¿se integraría con el resto del sistema?

### 7. Reflexión final

Pregunta al alumno: ¿usar el nombre del patrón en el prompt mejoró la calidad del código?, ¿la IA implementó correctamente todos los patrones?, ¿cómo usarías esto para acelerar el diseño de software en tu trabajo?

## Reglas de formato

NO uses la palabra "minuto". Estructura el ejercicio en "Paso 1", "Paso 2", etc.

El ejercicio debe ocupar al alumno aproximadamente 15 minutos.

Los prompts deben estar en bloques de código Markdown.

Incluye espacios para pegar el código generado y la tabla de evaluación.

Los prompts deben ser genéricos y funcionar en cualquier IA.

## Tono

Profesional pero cercano. Enfatiza que los patrones de diseño son un atajo semántico que la IA entiende perfectamente.
