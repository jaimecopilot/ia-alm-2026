# P4.3 – REFACTORIZACIÓN APLICANDO CLEAN CODE CON IA

Actúa como un diseñador instruccional experto en formación de desarrolladores de software. Necesito que generes un ejercicio práctico detallado para un curso de IA aplicada al desarrollo de software.

## Contexto del módulo

El módulo es "IA en Desarrollo y Codificación". Los alumnos están aprendiendo a usar IAs para refactorizar código aplicando principios de Clean Code.

## Concepto clave a transmitir

La refactorización con IA no es solo pedir "mejora este código". Consiste en describir exactamente qué principios aplicar (nombres descriptivos, extracción de métodos, eliminación de duplicidades, SRP) y proporcionar restricciones (mantener la interfaz pública, no cambiar el comportamiento). La IA ejecuta la transformación mecánica; el desarrollador valida que el comportamiento no ha cambiado.

## Objetivo del ejercicio

Que los alumnos aprendan a refactorizar código con malas prácticas aplicando principios de Clean Code mediante prompts específicos, y a verificar que el comportamiento no cambia ejecutando tests.

## Escenario

Proporciona un fragmento de código con malas prácticas intencionadas: nombres crípticos (f, x, y, z), funciones con múltiples responsabilidades, comentarios inútiles, falta de type hints, clases sin docstrings.

## Estructura que debes generar

### 1. Introducción

Explica los principios de Clean Code que se van a aplicar: nombres descriptivos, funciones pequeñas, SRP, DRY, type hints, docstrings.

### 2. Prompt 1: Renombrar y añadir type hints

Proporciona el código legacy y un prompt para que el alumno pida a la IA que renombre todas las variables y funciones con nombres descriptivos y añada type hints.

### 3. Prompt 2: Extraer métodos y aplicar SRP

Proporciona un prompt para que la IA divida las funciones largas en métodos más pequeños, cada uno con una única responsabilidad.

### 4. Prompt 3: Añadir docstrings y eliminar duplicidades

Proporciona un prompt para añadir docstrings en formato Google y eliminar código duplicado.

### 5. Prompt 4: Generar tests de caracterización

Proporciona un prompt para que la IA genere tests que capturen el comportamiento actual del código antes de refactorizar, y los ejecute después para verificar que la refactorización no rompió nada.

### 6. Comparación antes/después

Incluye espacio para que el alumno compare el código original con el refactorizado y evalúe la mejora en legibilidad y mantenibilidad.

### 7. Reflexión final

Pregunta: ¿qué principio de Clean Code tuvo mayor impacto?, ¿los tests de caracterización detectaron algún cambio de comportamiento?, ¿cómo incorporarías la refactorización asistida por IA en tu flujo de trabajo?

## Reglas de formato

NO uses la palabra "minuto". Estructura el ejercicio en "Paso 1", "Paso 2", etc.

El ejercicio debe ocupar al alumno aproximadamente 15 minutos.

Los prompts deben estar en bloques de código Markdown.

Incluye el código legacy a refactorizar y espacios para el código refactorizado.

Los prompts deben ser genéricos y funcionar en cualquier IA.

## Tono

Profesional pero cercano.
