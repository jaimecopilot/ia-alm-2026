# P4.5 – EXPLICACIÓN DE CÓDIGO COMPLEJO LÍNEA POR LÍNEA

Actúa como un diseñador instruccional experto en formación de desarrolladores de software. Necesito que generes un ejercicio práctico detallado para un curso de IA aplicada al desarrollo de software.

## Contexto del módulo

El módulo es "IA en Desarrollo y Codificación". Los alumnos están aprendiendo a usar IAs para entender código complejo o desconocido, una habilidad esencial para trabajar con código legacy o generado por IA.

## Concepto clave a transmitir

La IA no solo genera código: también puede explicarlo. Entender código complejo es tan importante como escribirlo. La IA puede actuar como un tutor que explica línea por línea, identifica la lógica de negocio subyacente y señala posibles problemas. Esta capacidad es fundamental en el ALM generativo, donde el desarrollador debe validar código que no ha escrito manualmente.

## Objetivo del ejercicio

Que los alumnos aprendan a usar IAs para entender código complejo, obtener explicaciones a diferentes niveles de profundidad (desde alto nivel hasta línea por línea), generar documentación a partir del código e identificar posibles bugs o mejoras.

## Escenario

Proporciona un fragmento de código intencionadamente complejo. Debe incluir: estructuras anidadas (defaultdict), lambdas, comprensiones de listas, lógica de negocio no evidente a simple vista, imports no utilizados (pistas falsas) y un bug sutil.

## Estructura que debes generar

### 1. Introducción

Explica que entender código legacy o generado por IA es una habilidad crítica. La IA puede ser tu tutor personal para desentrañar código complejo.

### 2. Prompt 1: Explicación general de alto nivel

Proporciona el código complejo y un prompt para que la IA explique qué hace a alto nivel, sin entrar en detalles sintácticos. El alumno debe obtener una visión general del propósito del código.

### 3. Prompt 2: Explicación línea por línea

Proporciona un prompt para que la IA explique cada línea individualmente, describiendo qué hace, por qué está ahí y cómo contribuye al resultado final. El alumno debe obtener una comprensión detallada.

### 4. Prompt 3: Explicación para diferentes audiencias

Proporciona un prompt para que la IA explique el mismo código a dos audiencias diferentes: un desarrollador junior (con más detalle técnico) y un stakeholder no técnico (con analogías de negocio). El alumno debe comparar ambas explicaciones.

### 5. Prompt 4: Identificar problemas y sugerir mejoras

Proporciona un prompt para que la IA analice el código en busca de: bugs potenciales, edge cases no manejados, imports no utilizados, código inalcanzable, problemas de rendimiento y sugerencias de refactorización.

### 6. Prompt 5: Generar documentación a partir del código

Proporciona un prompt para que la IA genere docstrings, un README explicativo del módulo y un diagrama Mermaid del flujo de la función principal.

### 7. Tabla de evaluación

Crea una tabla con columnas para cada tipo de explicación (alto nivel, línea por línea, junior, stakeholder) y filas para: claridad, precisión técnica, adaptación a la audiencia, utilidad práctica.

### 8. Reflexión final

Pregunta al alumno: ¿qué nivel de explicación te resultó más útil?, ¿la IA identificó correctamente el bug y los edge cases?, ¿cómo usarías esta capacidad para incorporarte a un proyecto con código legacy?, ¿confiarías en la explicación de la IA sin verificarla?

## Reglas de formato

NO uses la palabra "minuto". Estructura el ejercicio en "Paso 1", "Paso 2", etc.

El ejercicio debe ocupar al alumno aproximadamente 15 minutos.

Los prompts deben estar en bloques de código Markdown.

Incluye el código complejo a analizar y espacios para las explicaciones generadas.

Los prompts deben ser genéricos y funcionar en cualquier IA.

## Tono

Profesional pero cercano. Enfatiza que entender el código generado por IA es tan importante como generarlo.
