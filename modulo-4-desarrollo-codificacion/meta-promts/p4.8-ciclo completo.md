# P4.8 – CICLO COMPLETO: DEL PROMPT EJECUTABLE AL CÓDIGO VALIDADO

Actúa como un diseñador instruccional experto en formación de desarrolladores de software. Necesito que generes un ejercicio práctico detallado para un curso de IA aplicada al desarrollo de software.

## Contexto del módulo

El módulo es "IA en Desarrollo y Codificación". Este es el ejercicio integrador que cierra el módulo, combinando todas las habilidades aprendidas: generación de código, refactorización, explicación, detección de bugs y aplicación de patrones.

## Concepto clave a transmitir

El ALM generativo es un flujo continuo: especificación ejecutable → generación de código → validación → tests → refactorización → documentación. El desarrollador no escribe cada línea, sino que dirige agentes de IA en cada fase, supervisando y validando el resultado. Este ejercicio simula ese flujo completo.

## Objetivo del ejercicio

Que los alumnos ejecuten el ciclo completo de desarrollo con IA: partir de una especificación ejecutable, generar el código, validarlo, escribir tests, refactorizarlo aplicando Clean Code, detectar y corregir bugs, y generar documentación.

## Escenario

Proporciona una especificación ejecutable completa para una funcionalidad (puede ser una nueva o una de las generadas en ejercicios anteriores). La funcionalidad debe ser lo suficientemente compleja como para requerir múltiples archivos y decisiones de diseño.

## Estructura que debes generar

### 1. Introducción

Explica el flujo completo del ALM generativo en la fase de desarrollo. El desarrollador es el director de orquesta.

### 2. Paso 1: Generar el código desde la especificación

Proporciona un prompt para que el alumno use la especificación ejecutable como prompt y obtenga el código multi-archivo de la IA.

### 3. Paso 2: Validar el código generado

Proporciona un checklist de validación: revisar imports, verificar tipos, comprobar lógica, ejecutar tests.

### 4. Paso 3: Generar y ejecutar tests

Proporciona un prompt para generar tests unitarios y de integración, ejecutarlos y verificar que pasan.

### 5. Paso 4: Refactorizar aplicando Clean Code

Proporciona un prompt para refactorizar el código generado: mejorar nombres, extraer métodos, añadir docstrings, eliminar duplicidades.

### 6. Paso 5: Detectar y corregir bugs

Proporciona un prompt para que la IA analice el código refactorizado en busca de posibles bugs o edge cases no cubiertos, y los corrija.

### 7. Paso 6: Generar documentación

Proporciona un prompt para generar docstrings, README del módulo y un diagrama Mermaid de la arquitectura del código generado.

### 8. Producto final

Indica al alumno que debe entregar: código generado, código refactorizado, tests, documentación y un breve informe de lecciones aprendidas.

### 9. Reflexión final

Pregunta al alumno: ¿cómo cambió tu percepción del desarrollo con IA después de este ciclo completo?, ¿qué fase requirió más intervención humana?, ¿qué fase fue más automatizable?, ¿cómo aplicarías este flujo en tu trabajo diario?

## Reglas de formato

NO uses la palabra "minuto". Estructura el ejercicio en "Paso 1", "Paso 2", etc.

El ejercicio debe ocupar al alumno aproximadamente 15 minutos.

Los prompts deben estar en bloques de código Markdown.

Incluye espacios para cada entregable y la reflexión final.

Los prompts deben ser genéricos y funcionar en cualquier IA.

## Tono

Profesional pero cercano. Enfatiza que este flujo es el presente del desarrollo de software, no el futuro.
