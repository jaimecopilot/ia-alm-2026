# P4.2 – CREACIÓN DE UN CRUD API REST COMPLETO DESDE PROMPT EJECUTABLE

Actúa como un diseñador instruccional experto en formación de desarrolladores de software. Necesito que generes un ejercicio práctico detallado para un curso de IA aplicada al desarrollo de software.

## Contexto del módulo

El módulo es "IA en Desarrollo y Codificación". Los alumnos están aprendiendo a generar código multi-archivo a partir de especificaciones ejecutables.

## Concepto clave a transmitir

Una especificación ejecutable bien redactada puede generar no solo una función, sino un CRUD completo con modelos, schemas, repositorios, servicios, endpoints y tests. Este es el flujo del ALM generativo: el requisito se convierte directamente en código funcional multi-archivo.

## Objetivo del ejercicio

Que los alumnos aprendan a redactar una especificación ejecutable para un CRUD y usarla para generar todo el código necesario (modelo, schema, repositorio, servicio, endpoints y tests) de una sola vez.

## Escenario

Proporciona el contexto: "Sistema de reservas de salas. Necesitamos un CRUD completo para gestionar las salas de reuniones".

## Estructura que debes generar

### 1. Introducción

Explica el concepto de especificación ejecutable para generación multi-archivo. Una buena especificación debe incluir: entidad con atributos, endpoints con métodos HTTP, validaciones, patrones a usar y criterios de aceptación.

### 2. Prompt: Especificación ejecutable para CRUD

Proporciona un prompt de ejemplo que el alumno puede adaptar. Debe incluir: entidad (Sala) con todos sus atributos y tipos, endpoints (GET all, GET by id, POST, PUT, DELETE) con códigos de respuesta, validaciones, patrón Repository-Service, framework (FastAPI), ORM (SQLAlchemy) y criterios Gherkin.

### 3. Instrucciones para generar el código

Indica al alumno que pegue la especificación en al menos 2 IAs diferentes y compare el código generado. Debe evaluar: ¿generó todos los archivos necesarios?, ¿los imports son correctos?, ¿las validaciones coinciden con lo especificado?, ¿los tests cubren los criterios Gherkin?

### 4. Tabla de evaluación

Crea una tabla con columnas para cada IA y filas para cada artefacto (modelo, schema, repositorio, servicio, endpoints, tests). Pide al alumno que evalúe cada uno del 1 al 5.

### 5. Reflexión final

Pregunta: ¿qué IA generó el código más completo?, ¿faltó algún archivo o validación?, ¿el código generado podría desplegarse directamente?, ¿qué añadirías a la especificación para mejorar el resultado?

## Reglas de formato

NO uses la palabra "minuto". Estructura el ejercicio en "Paso 1", "Paso 2", etc.

El ejercicio debe ocupar al alumno aproximadamente 15 minutos.

Los prompts deben estar en bloques de código Markdown.

Incluye espacios para pegar el código generado y la tabla de evaluación.

Los prompts deben ser genéricos y funcionar en cualquier IA.

## Tono

Profesional pero cercano. Enfatiza que una buena especificación es el factor determinante para obtener código de calidad.
