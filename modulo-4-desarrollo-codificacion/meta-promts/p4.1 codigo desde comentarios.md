# P4.1 – GENERACIÓN DE CÓDIGO DESDE COMENTARIOS Y ESPECIFICACIONES

Actúa como un diseñador instruccional experto en formación de desarrolladores de software. Necesito que generes un ejercicio práctico detallado para un curso de IA aplicada al desarrollo de software.

## Contexto del módulo

El módulo es "IA en Desarrollo y Codificación". Los alumnos son desarrolladores que están aprendiendo a generar código usando IAs a partir de comentarios descriptivos y especificaciones ejecutables.

## Concepto clave a transmitir

La calidad del código generado por IA depende directamente de la calidad del prompt. Un comentario vago produce código genérico. Un comentario que especifica parámetros, validaciones, formato de retorno y casos límite produce código profesional. En el contexto del ALM generativo, el comentario o la especificación actúa como el prompt que dispara la generación de código funcional.

## Objetivo del ejercicio

Que los alumnos experimenten cómo la precisión del comentario determina la calidad del código generado, usando tres niveles de detalle: comentario vago, comentario detallado y especificación ejecutable completa.

## Escenario

Proporciona un contexto: "Sistema de reservas de salas. Necesitamos una función que verifique si una sala está disponible en una franja horaria".

## Estructura que debes generar

### 1. Introducción

Explica que el comentario es un prompt. Cuanto más preciso sea, mejor será el código generado.

### 2. Prompt 1: Comentario vago

Proporciona un prompt con un comentario minimalista: "Función que verifica disponibilidad de una sala". Pide al alumno que lo use en al menos 2 IAs y evalúe el resultado.

### 3. Prompt 2: Comentario detallado

Proporciona un prompt con un comentario que incluye: propósito, parámetros con tipos, lógica de solapamiento, validaciones y formato de retorno. Pide al alumno que compare el código generado con el del Prompt 1.

### 4. Prompt 3: Especificación ejecutable completa

Proporciona un prompt con una especificación que incluye: contexto de negocio, detalles técnicos (framework, ORM, archivos), criterios Gherkin y restricciones. Pide al alumno que evalúe si este código podría integrarse directamente en un proyecto real.

### 5. Tabla comparativa

Crea una tabla con columnas para cada nivel y filas para: type hints, docstring, validaciones, manejo de errores, legibilidad, integrabilidad en un proyecto real.

### 6. Reflexión final

Pregunta al alumno: ¿qué nivel produjo código listo para producción?, ¿cuánto tiempo extra lleva escribir un buen comentario frente a arreglar código mediocre?, ¿cómo aplicarías esto en tu día a día?

## Reglas de formato

NO uses la palabra "minuto". Estructura el ejercicio en "Paso 1", "Paso 2", etc.

El ejercicio debe ocupar al alumno aproximadamente 15 minutos.

Los prompts deben estar en bloques de código Markdown.

Incluye espacios para pegar el código generado y la tabla comparativa.

Los prompts deben ser genéricos y funcionar en cualquier IA de codificación.

## Tono

Profesional pero cercano. Enfatiza que el tiempo invertido en un buen prompt se recupera con creces.
