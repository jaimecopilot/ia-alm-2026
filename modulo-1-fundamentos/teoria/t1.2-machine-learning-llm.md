# T1.2 – Conceptos básicos de aprendizaje automático y modelos de lenguaje

En la programación tradicional, tú escribes las reglas. Defines cada condición, cada bucle, cada caso de uso. El ordenador ejecuta exactamente lo que has programado. Si quieres que un sistema reconozca si un correo es spam, escribes reglas como "si contiene la palabra 'viagra', clasifícalo como spam". Pero el lenguaje humano es variable y siempre aparecerán nuevos patrones que tus reglas no cubren. El aprendizaje automático invierte ese planteamiento: proporcionas muchos ejemplos etiquetados y dejas que el sistema descubra por sí mismo qué patrones distinguen una categoría de otra.

Existen tres grandes tipos de aprendizaje automático. El **aprendizaje supervisado** entrena al modelo con ejemplos que incluyen tanto la entrada como la salida esperada. Si quieres que un modelo aprenda a traducir entre lenguajes de programación, le muestras pares de funciones equivalentes en Java y Python. El **aprendizaje no supervisado** busca patrones en datos sin etiquetar, como cuando los modelos aprenden la estructura del lenguaje durante el pre-entrenamiento. El **aprendizaje por refuerzo** entrena al modelo mediante recompensas: genera una respuesta, un evaluador la puntúa, y el modelo ajusta su comportamiento para maximizar la puntuación. Este último es el método usado para alinear modelos como ChatGPT con las preferencias humanas.

El **deep learning** utiliza redes neuronales artificiales compuestas por capas de neuronas. Cada neurona recibe valores numéricos de entrada, les aplica un peso —un número que indica la importancia de esa entrada—, los suma y aplica una función de activación que decide si la neurona se "dispara" y transmite información a la capa siguiente. Durante el entrenamiento, la red ajusta esos pesos para minimizar el error en sus predicciones. Los modelos de lenguaje modernos tienen cientos de capas y miles de millones de parámetros.

Un modelo de lenguaje es, en esencia, un **predictor del siguiente token**. Un token puede ser una palabra completa, parte de una palabra, un signo de puntuación o un fragmento de código. El modelo recibe una secuencia de tokens —el contexto— y calcula, para cada posible token de su vocabulario, la probabilidad de que sea la continuación correcta. La generación es incremental: predice el primer token, lo añade al contexto, predice el segundo, y así sucesivamente. Este proceso se llama **generación autoregresiva**.

La precisión de cualquier asistente de IA depende de la cantidad y calidad del contexto que recibe. Cuando escribes código en un IDE con un asistente integrado, este no solo lee la línea actual: analiza el archivo completo, otras pestañas abiertas, los imports y los comentarios cercanos. Toda esa información se convierte en tokens y se introduce en el modelo. Por eso un comentario bien escrito puede cambiar radicalmente la sugerencia. Si escribes `# Función que valida un email`, el modelo asocia esos tokens con patrones de validación de emails y te propone una implementación coherente.

Entender cómo aprende y funciona un modelo de lenguaje no es solo curiosidad técnica. Te ayuda a comprender por qué los asistentes aciertan más cuando les das buen contexto, por qué a veces fallan en tareas muy específicas y, sobre todo, por qué es tan importante saber comunicarte con ellos mediante prompts eficaces.
