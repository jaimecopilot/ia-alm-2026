# P3.1 – EXPLORAR Y COMPARAR ARQUITECTURAS CON IA

Actúa como un diseñador instruccional experto en formación de desarrolladores de software. Necesito que generes un ejercicio práctico detallado para un curso de IA aplicada al desarrollo de software.

## Contexto del módulo

El módulo es "IA en Diseño de Software". Los alumnos son desarrolladores que están aprendiendo a usar IAs generativas como sparring arquitectónico para explorar y comparar arquitecturas.

## Concepto clave a transmitir

La IA no decide por ti, pero puede actuar como un compañero de diseño que genera alternativas, evalúa trade-offs y te ayuda a pensar en las implicaciones a largo plazo de cada decisión arquitectónica. El objetivo es usar la IA para expandir tu abanico de opciones, no para delegar la decisión.

## Objetivo del ejercicio

Que los alumnos aprendan a usar IAs para explorar arquitecturas alternativas para un mismo sistema, compararlas de forma estructurada y seleccionar la más adecuada justificando la decisión.

## Escenario

Proporciona una especificación ejecutable de un sistema (puede ser la generada en el Módulo 2 o una nueva). El sistema debe tener suficientes requisitos como para que la decisión arquitectónica no sea trivial: por ejemplo, un sistema de reservas con notificaciones, autenticación, informes y alta concurrencia esperada.

## Estructura que debes generar

### 1. Introducción

Explica el concepto de sparring arquitectónico. La IA es un compañero de diseño, no un oráculo.

### 2. Prompt 1: Explorar arquitecturas alternativas

Proporciona un prompt para que el alumno pida a la IA que proponga al menos 3 arquitecturas diferentes para el sistema (ej: monolito modular, microservicios, arquitectura hexagonal con eventos). Para cada una, debe pedir: descripción general, componentes principales, ventajas, desventajas y casos de uso ideales.

### 3. Prompt 2: Comparación estructurada

Proporciona un prompt para que el alumno pida a la IA que compare las arquitecturas usando criterios específicos: escalabilidad, complejidad operativa, velocidad de desarrollo, coste de infraestructura, mantenibilidad y adecuación al equipo.

### 4. Prompt 3: Simular evolución futura

Proporciona un prompt para que el alumno pida a la IA que simule cómo evolucionaría cada arquitectura si el sistema crece (ej: de 500 a 5000 usuarios, de 3 a 15 desarrolladores).

### 5. Tabla comparativa

Crea una tabla con columnas para cada arquitectura y filas para cada criterio. Pide al alumno que la complete con las respuestas de la IA y añada su propia valoración.

### 6. Reflexión final

Pregunta al alumno: ¿qué arquitectura elegirías y por qué?, ¿la IA mencionó factores que no habías considerado?, ¿en qué aspectos la recomendación de la IA coincidió con tu intuición y en cuáles difirió?, ¿cómo usarías este enfoque en tu trabajo diario?

## Reglas de formato

- NO uses la palabra "minuto".
- Estructura el ejercicio en "Paso 1", "Paso 2", etc.
- El ejercicio debe ocupar al alumno aproximadamente 15 minutos.
- Los prompts deben estar en bloques de código Markdown.
- Incluye espacios para respuestas y la tabla comparativa.
- Los prompts deben ser genéricos y funcionar en cualquier IA.

## Tono

Profesional pero cercano. Enfatiza que la IA es una herramienta de exploración, no un sustituto del criterio humano.
