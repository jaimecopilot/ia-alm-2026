# P3.2 – GENERAR DIAGRAMAS C4 Y UML DESDE ESPECIFICACIONES

Actúa como un diseñador instruccional experto en formación de desarrolladores de software. Necesito que generes un ejercicio práctico detallado para un curso de IA aplicada al desarrollo de software.

## Contexto del módulo

El módulo es "IA en Diseño de Software". Los alumnos están aprendiendo a generar diagramas de arquitectura C4 y UML usando IAs, partiendo de especificaciones ejecutables.

## Concepto clave a transmitir

Los diagramas generados por IA en formato Mermaid o PlantUML no son imágenes estáticas: son texto versionable que evoluciona con el código. Pueden generarse desde especificaciones, refinarse conversacionalmente y regenerarse cuando la arquitectura cambia. El diagrama es un subproducto vivo del diseño, no un artefacto estático.

## Objetivo del ejercicio

Que los alumnos aprendan a generar diagramas C4 (contexto y contenedores) y diagramas UML (clases y secuencia) a partir de una especificación ejecutable, refinarlos iterativamente y versionarlos.

## Escenario

Proporciona una especificación ejecutable de un sistema de reservas (la misma del ejercicio anterior o una nueva).

## Estructura que debes generar

### 1. Introducción

Explica el concepto de diagramas como código. Mermaid y PlantUML permiten definir diagramas con texto. La IA puede generar ese texto.

### 2. Prompt 1: Generar diagrama C4 de contexto

Proporciona un prompt para que el alumno pida a la IA que genere un diagrama C4 de contexto en Mermaid basado en la especificación. Debe mostrar actores, sistemas externos y el sistema principal con sus relaciones.

### 3. Prompt 2: Generar diagrama C4 de contenedores

Proporciona un prompt para generar el nivel de contenedores: aplicaciones, bases de datos, servicios que componen el sistema, con sus relaciones y protocolos de comunicación.

### 4. Prompt 3: Generar diagrama de clases UML

Proporciona un prompt para generar un diagrama de clases con las entidades del dominio, sus atributos, métodos y relaciones.

### 5. Prompt 4: Generar diagrama de secuencia

Proporciona un prompt para generar un diagrama de secuencia para uno de los flujos principales descritos en la especificación.

### 6. Instrucciones de refinamiento

Explica al alumno cómo refinar cada diagrama: pedir a la IA que añada, modifique o elimine elementos hasta que el diagrama refleje fielmente la arquitectura deseada.

### 7. Tabla de evaluación

Crea una tabla para que el alumno evalúe cada diagrama: ¿refleja correctamente la especificación?, ¿es completo?, ¿es claro?, ¿podría generarse código a partir de él?

### 8. Reflexión final

Pregunta: ¿los diagramas generados reflejan fielmente los requisitos?, ¿qué ventajas tiene el diagrama como código frente a una imagen estática?, ¿cómo mantenerías estos diagramas actualizados a medida que el sistema evoluciona?

## Reglas de formato

- NO uses la palabra "minuto".
- Estructura el ejercicio en "Paso 1", "Paso 2", etc.
- El ejercicio debe ocupar al alumno aproximadamente 15 minutos.
- Los prompts deben estar en bloques de código Markdown.
- Incluye espacios para pegar el código Mermaid generado y para la tabla de evaluación.
- Los prompts deben ser genéricos y funcionar en cualquier IA.

## Tono

Profesional pero cercano. Enfatiza la importancia de la iteración para refinar los diagramas.
