# P3.5 – DOCUMENTAR DECISIONES CON ADRs GENERADOS POR IA

Actúa como un diseñador instruccional experto en formación de desarrolladores de software. Necesito que generes un ejercicio práctico detallado para un curso de IA aplicada al desarrollo de software.

## Contexto del módulo

El módulo es "IA en Diseño de Software". Los alumnos están aprendiendo a documentar decisiones arquitectónicas usando Architecture Decision Records (ADRs) generados por IA.

## Concepto clave a transmitir

Cada decisión arquitectónica debe documentarse: qué se decidió, por qué, qué alternativas se consideraron y qué consecuencias tendrá. Los ADRs son el formato estándar para esta documentación. La IA puede generarlos a partir de la conversación de diseño, asegurando trazabilidad.

## Objetivo del ejercicio

Que los alumnos aprendan a generar ADRs para las decisiones arquitectónicas tomadas durante el diseño, usando IAs para estructurar y redactar el documento.

## Escenario

Basado en los ejercicios anteriores, el alumno ha tomado varias decisiones: arquitectura (monolito/microservicios), base de datos (SQL/NoSQL), autenticación (JWT/OAuth), comunicación entre servicios (REST/eventos).

## Estructura que debes generar

1. Introducción  
Explica qué es un ADR, su estructura estándar (contexto, decisión, alternativas, consecuencias) y por qué es importante documentar decisiones.

2. Prompt 1: Generar ADR para la decisión de arquitectura  
Proporciona un prompt para que el alumno pida a la IA que genere un ADR documentando la elección de arquitectura (monolito vs microservicios), basado en la conversación de diseño.

3. Prompt 2: Generar ADR para la decisión de base de datos  
Proporciona un prompt similar para la elección de base de datos.

4. Prompt 3: Generar ADR para la decisión de comunicación  
Proporciona un prompt para documentar la elección del patrón de comunicación entre servicios.

5. Prompt 4: Generar índice de ADRs  
Proporciona un prompt para que la IA genere un índice Markdown con todos los ADRs, incluyendo una breve descripción de cada uno.

6. Tabla de evaluación  
Crea una tabla para evaluar cada ADR: ¿incluye contexto?, ¿la decisión está claramente expresada?, ¿las alternativas están bien analizadas?, ¿las consecuencias son realistas?

7. Reflexión final  
Pregunta: ¿qué valor aporta documentar las decisiones?, ¿cómo ayudan los ADRs a nuevos miembros del equipo?, ¿cómo mantendrías los ADRs actualizados?

## Reglas de formato

NO uses la palabra "minuto". Estructura el ejercicio en "Paso 1", "Paso 2", etc.

El ejercicio debe ocupar al alumno aproximadamente 15 minutos.

Los prompts deben estar en bloques de código Markdown.

Incluye espacios para pegar los ADRs generados y la tabla de evaluación.

Los prompts deben ser genéricos y funcionar en cualquier IA.

## Tono  
Profesional pero cercano.
