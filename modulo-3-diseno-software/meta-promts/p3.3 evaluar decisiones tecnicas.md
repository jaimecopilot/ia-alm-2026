# P3.3 – EVALUAR DECISIONES TÉCNICAS CON MÚLTIPLES IAS

Actúa como un diseñador instruccional experto en formación de desarrolladores de software. Necesito que generes un ejercicio práctico detallado para un curso de IA aplicada al desarrollo de software.

## Contexto del módulo

El módulo es "IA en Diseño de Software". Los alumnos están aprendiendo a evaluar decisiones técnicas usando IAs como comité de evaluación, comparando análisis de diferentes modelos.

## Concepto clave a transmitir

Diferentes IAs pueden dar diferentes recomendaciones para la misma decisión técnica. Comparar sus análisis permite identificar sesgos y obtener una visión más completa. La decisión final debe estar justificada y documentada en un ADR.

## Objetivo del ejercicio

Que los alumnos aprendan a plantear la misma decisión técnica a múltiples IAs, comparar sus análisis y generar un ADR documentando la decisión final.

## Escenario

Proporciona un contexto de sistema y plantea una decisión técnica concreta. Por ejemplo: "Para un sistema de reservas con 500 empleados y 3 desarrolladores, ¿recomiendas monolito modular con FastAPI o microservicios desde el inicio?"

## Estructura que debes generar

### 1. Introducción

Explica que diferentes IAs tienen diferentes sesgos y fortalezas. Comparar sus análisis da una visión más completa.

### 2. Prompt 1: Evaluación con ChatGPT

Proporciona un prompt para que el alumno pida a ChatGPT que analice la decisión desde múltiples perspectivas (técnica, organizacional, económica).

### 3. Prompt 2: Evaluación con Claude

Proporciona el mismo prompt para Claude. Pide al alumno que compare las respuestas: ¿qué factores menciona uno que el otro no?, ¿qué diferencias de tono y profundidad hay?

### 4. Prompt 3: Evaluación con Gemini

Proporciona el mismo prompt para Gemini. Pide al alumno que compare las tres respuestas.

### 5. Tabla comparativa

Crea una tabla con columnas para cada IA y filas para cada factor considerado. Pide al alumno que marque qué factores mencionó cada una.

### 6. Prompt 4: Generar ADR

Proporciona un prompt para que el alumno pida a la IA que genere un Architecture Decision Record documentando la decisión final, incluyendo contexto, alternativas consideradas y consecuencias.

### 7. Reflexión final

Pregunta: ¿hubo consenso entre las IAs?, ¿alguna mencionó factores que no habías considerado?, ¿cómo afecta saber que diferentes IAs pueden recomendar cosas diferentes?

## Reglas de formato

- NO uses la palabra "minuto".
- Estructura el ejercicio en "Paso 1", "Paso 2", etc.
- El ejercicio debe ocupar al alumno aproximadamente 15 minutos.
- Los prompts deben estar en bloques de código Markdown.
- Incluye espacios para respuestas y la tabla comparativa.
- Los prompts deben ser genéricos y funcionar en cualquier IA.

## Tono

Profesional pero cercano. Enfatiza que la comparación entre IAs reduce el riesgo de sesgo.
