Actúa como un arquitecto de ALM Generativo experto en testing con asistentes de IDE. Estás diseñando un laboratorio práctico para desarrolladores que quieren usar asistentes y agentes de IDE (Copilot Chat, Cursor, Codeium) para generar, ejecutar y reparar tests.

## Contexto del ALM Generativo

En el ALM generativo, el testing no es una fase separada: los tests se generan junto con el código, se ejecutan automáticamente y los fallos se reparan en un bucle de auto-reparación. Los asistentes de IDE son la primera línea de este proceso, permitiendo generar tests con comandos slash y reparar fallos con Agent Mode.

## Tu tarea

Genera un laboratorio guiado "Paso a Paso" que el alumno pueda ejecutar con Copilot Chat, Cursor o Codeium.

El laboratorio debe cubrir:

### Visión en el ALM
Explica que el testing en el ALM generativo es un bucle continuo de generación, ejecución y reparación.

### Técnica 1: Generar tests unitarios con /tests
Guía al alumno para que seleccione una función y use el comando /tests para generar una suite completa.

### Técnica 2: Generar tests de integración
Guía al alumno para que pida tests de integración para un endpoint, con base de datos de prueba y cliente HTTP.

### Técnica 3: Reparar tests fallidos con /fixTestFailure
Guía al alumno para que provoque un fallo y use el comando /fixTestFailure para repararlo.

### Técnica 4: Agent Mode para auto-reparación
Guía al alumno para que active Agent Mode y pida al agente que ejecute los tests y corrija fallos autónomamente.

### Técnica 5: Mejorar cobertura
Guía al alumno para que analice la cobertura y pida tests adicionales para líneas no cubiertas.

## Conexión con el resto del ALM

Pide al alumno que documente cómo los tests generados se integran en el pipeline de CI/CD.
