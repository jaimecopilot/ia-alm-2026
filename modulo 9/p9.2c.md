Estás diseñando un laboratorio práctico para desarrolladores que quieren usar agentes autónomos (Copilot Workspace, Azure DevOps + Copilot) para transformar requisitos en planes de implementación y Pull Requests.

## Contexto del ALM Generativo

Los agentes autónomos representan la culminación del ALM generativo en la fase de análisis: no solo generan requisitos, sino que los transforman directamente en planes de implementación y código. Un issue bien redactado en GitHub o un Work Item en Azure DevOps se convierte en el prompt que dispara todo el flujo: planificación → implementación → tests → PR. El desarrollador pasa de escribir requisitos a revisar planes generados por IA.

## Tu tarea

Genera un laboratorio guiado "Paso a Paso" que el alumno pueda ejecutar con Copilot Workspace o la integración Azure DevOps + Copilot.

El laboratorio debe cubrir:

### Visión en el ALM
Explica que los agentes autónomos cierran el círculo entre análisis e implementación. Un requisito bien redactado se convierte automáticamente en un plan de implementación y, posteriormente, en código.

### Técnica 1: Crear un issue como prompt ejecutable
Guía al alumno para que redacte un issue de GitHub con todos los elementos necesarios: título descriptivo, contexto, solución propuesta, detalles técnicos, criterios Gherkin y restricciones.

### Técnica 2: Abrir en Copilot Workspace
Guía al alumno para que abra el issue en Copilot Workspace y observe cómo el agente genera un plan de implementación detallado, analizando el repositorio.

### Técnica 3: Interactuar con el Ask Questions Tool
Guía al alumno para que responda a las preguntas del agente durante la fase de alineación, clarificando decisiones de diseño.

### Técnica 4: Revisar y refinar el plan
Guía al alumno para que revise el plan generado, solicite cambios ("añade validación de fechas", "separa en dos servicios") y observe cómo el plan se actualiza.

### Técnica 5: Aprobar y ejecutar
Guía al alumno para que apruebe el plan y observe cómo el agente implementa el código en múltiples archivos, ejecuta tests y genera una PR.

## Conexión con el resto del ALM

Pide al alumno que documente cómo la PR generada se conecta con las fases de testing (CI/CD) y despliegue, cerrando el círculo del ALM generativo.
