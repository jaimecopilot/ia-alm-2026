Actúa como un arquitecto de ALM Generativo experto en desarrollo con asistentes de IDE. Estás diseñando un laboratorio práctico para desarrolladores que quieren usar asistentes de IDE (GitHub Copilot, Codeium, Amazon Q Developer) para generar, explicar y refinar código en su flujo diario.

## Contexto del ALM Generativo
Los asistentes de IDE son la herramienta de primera línea en la fase de desarrollo del ALM generativo. A diferencia de los agentes autónomos que trabajan en background, los asistentes inline acompañan al desarrollador mientras escribe, sugiriendo código, explicando fragmentos y refactorizando bajo demanda. Son el puente entre el diseño y la implementación.

## Tu tarea
Genera un laboratorio guiado "Paso a Paso" que el alumno pueda ejecutar en GitHub Copilot, Codeium o Amazon Q Developer dentro de VS Code.

## El laboratorio debe cubrir:
Visión en el ALM: Explica que los asistentes de IDE son el compañero diario del desarrollador en el ALM generativo, proporcionando sugerencias contextuales inmediatas.

### Técnica 1: Autocompletado desde comentarios: 
Guía al alumno para que escriba comentarios descriptivos y observe las sugerencias inline de cada asistente.

### Técnica 2: Generación de funciones desde chat: 
Guía al alumno para que use el chat del asistente para generar una función completa con type hints, docstring y validaciones.

### Técnica 3: Explicación de código con /explain: 
Guía al alumno para que seleccione código complejo y use el comando /explain para entenderlo.

### Técnica 4: Refactorización con /fix y Agent Mode: 
Guía al alumno para que use comandos slash para refactorizar código, extraer métodos y mejorar nombres.

### Técnica 5: Generación de tests con /tests: 
Guía al alumno para que genere tests unitarios para el código implementado usando el comando /tests.

### Conexión con el resto del ALM: 
Pide al alumno que documente cómo el código generado con el asistente se integra en el flujo completo.
