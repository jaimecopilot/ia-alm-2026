# P9.1b – LABORATORIO GUIADO: AGENTES AUTÓNOMOS PARA DESARROLLO Y CODIFICACIÓN

## Visión en el ALM generativo

Los agentes autónomos representan la evolución del desarrollo asistido por IA. A diferencia de los asistentes de IDE, estos sistemas no solo sugieren código: planifican, ejecutan, prueban y entregan soluciones completas.

El desarrollador pasa de escribir código a supervisar resultados.

---

## Objetivo del laboratorio

En este laboratorio aprenderás a:

- Crear especificaciones ejecutables para agentes
- Ejecutar tareas autónomas de desarrollo
- Revisar planes de implementación generados por IA
- Validar código multiarchivo generado automáticamente
- Analizar Pull Requests creadas por agentes

---

## Paso 1 – Crear una especificación ejecutable

Una buena especificación es la clave del éxito del agente.

**Ejemplo de especificación:**

> Crear una API REST en Python con FastAPI que incluya:
> - CRUD de usuarios
> - Validación de email
> - Persistencia en SQLite
> - Tests unitarios
> - Dockerfile

**Ejercicio:**
Redacta una especificación similar para una aplicación de tu elección.

---

## Paso 2 – Ejecutar el agente

Carga la especificación en una de las siguientes herramientas:

- Copilot Workspace
- Cursor Agent
- Claude Code

Observa cómo el sistema:

- Analiza el repositorio
- Genera un plan de trabajo
- Divide la tarea en pasos

---

## Paso 3 – Revisar el plan generado

Antes de aprobar la ejecución:

- Verifica si cubre todos los requisitos
- Detecta omisiones o riesgos
- Solicita ajustes si es necesario

**Prompt sugerido:**

> “Revisa el plan y añade validación de datos avanzada y manejo de errores consistente”

---

## Paso 4 – Implementación autónoma

Una vez aprobado el plan:

- El agente modifica múltiples archivos
- Genera estructura de proyecto
- Implementa lógica de negocio
- Crea tests y configuración

Observa el flujo sin intervención directa.

---

## Paso 5 – Validación y auto-reparación

El agente ejecuta tests automáticamente.

Si hay errores:

- Detecta fallos
- Propone correcciones
- Reescribe partes del código

Este ciclo se conoce como **auto-reparación iterativa**.

---

## Paso 6 – Revisión de Pull Request

El agente genera una Pull Request con:

- Descripción automática
- Cambios detallados
- Resultados de tests

**Ejercicio:**
Analiza la PR y responde:

- ¿Es el código mantenible?
- ¿Cumple requisitos?
- ¿Qué mejorarías?

---

## Tabla comparativa de agentes

| Herramienta | Tipo | Fortalezas | Limitaciones |
|-------------|------|-------------|--------------|
| Copilot Workspace | Agente integrado | Integración GitHub | Menos flexible |
| Cursor Agent | IDE + agente | Alta autonomía | Dependencia del contexto |
| Claude Code | Agente LLM | Razonamiento profundo | Menos integrado |

---

## Conexión con el ALM generativo

Este laboratorio forma parte de la transición:

- De escribir código → a definir intenciones
- De funciones → a sistemas completos
- De ejecución manual → a supervisión

---

## Reflexión final

Los agentes autónomos no eliminan el control del desarrollador, pero sí eliminan gran parte del trabajo mecánico.

El valor se traslada a:

- Definición de requisitos
- Validación de calidad
- Decisión arquitectónica
