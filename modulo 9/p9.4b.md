## Herramientas cubiertas
Copilot Workspace, Cursor Agent, Claude Code

## Laboratorio generado por IA

Instrucción: El formador debe copiar este meta-prompt y pegarlo en ChatGPT, Claude o Gemini. La IA generará un laboratorio guiado completo listo para entregar al alumno.

Actúa como un arquitecto de ALM Generativo experto en desarrollo con agentes autónomos. Estás diseñando un laboratorio práctico para desarrolladores que quieren usar agentes autónomos (Copilot Workspace, Cursor Agent, Claude Code) para implementar código multi-archivo de forma autónoma.

## Contexto del ALM Generativo

Los agentes autónomos representan el siguiente nivel en el ALM generativo: no solo sugieren código, sino que planifican e implementan tareas completas que abarcan múltiples archivos. Toman una especificación o un issue, analizan el repositorio, generan un plan, implementan el código, ejecutan tests y abren una PR. El desarrollador pasa de escribir código a revisar y aprobar el trabajo del agente.

## Tu tarea

Genera un laboratorio guiado "Paso a Paso" que el alumno pueda ejecutar con Copilot Workspace, Cursor Agent o Claude Code.

El laboratorio debe cubrir:

### Visión en el ALM
Explica que los agentes autónomos son el motor del ALM generativo: transforman especificaciones en código funcional sin intervención humana continua.

### Técnica 1: Preparar una especificación ejecutable
Guía al alumno para que redacte una especificación detallada que servirá como prompt para el agente.

### Técnica 2: Ejecutar el agente
Guía al alumno para que abra la especificación en Copilot Workspace, Cursor Agent o Claude Code y observe cómo genera un plan de implementación.

### Técnica 3: Revisar el plan
Guía al alumno para que analice el plan generado, verifique que cubre todos los requisitos y solicite ajustes si es necesario.

### Técnica 4: Implementación autónoma
Guía al alumno para que apruebe el plan y observe cómo el agente implementa el código en múltiples archivos.

### Técnica 5: Verificación y tests
Guía al alumno para que observe cómo el agente ejecuta los tests y corrige fallos automáticamente (bucle de auto-reparación).

### Técnica 6: Revisar la PR generada
Guía al alumno para que revise la Pull Request generada por el agente, incluyendo la descripción automática y los resultados de los tests.

## Formato del laboratorio

NO uses la palabra "minuto". Estructura en "Paso 1", "Paso 2", etc.

Incluye prompts de ejemplo para la especificación.

Incluye una tabla comparativa de los agentes probados.

Incluye una reflexión final sobre el cambio de rol del desarrollador con agentes autónomos.
