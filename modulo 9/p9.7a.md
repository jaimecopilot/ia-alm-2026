Actúa como un arquitecto de ALM Generativo experto en mantenimiento con asistentes de IDE. Estás diseñando un laboratorio práctico para desarrolladores que quieren usar asistentes de IDE (Copilot Chat, Codeium, Amazon Q Developer) para analizar código legacy, generar documentación y refactorizar.

## Contexto del ALM Generativo
El mantenimiento en el ALM generativo no es una fase separada: es una actividad continua. Los asistentes de IDE permiten entender código legacy, documentarlo y refactorizarlo sin salir del editor, aprovechando el contexto del repositorio.

## Tu tarea
Genera un laboratorio guiado "Paso a Paso" que el alumno pueda ejecutar con Copilot Chat, Codeium o Amazon Q Developer.

## El laboratorio debe cubrir
Visión en el ALM: Explica que el mantenimiento con IA es continuo, no puntual.

Técnica 1: Explicar código legacy con /explain: Guía al alumno para que seleccione código complejo y use /explain.

Técnica 2: Generar documentación con /doc: Guía al alumno para que seleccione funciones y genere docstrings.

Técnica 3: Detectar code smells: Guía al alumno para que pida al asistente que analice el código en busca de problemas.

Técnica 4: Refactorizar con /fix y Agent Mode: Guía al alumno para que refactorice código aplicando Clean Code.

Técnica 5: Generar diagramas desde código: Guía al alumno para que genere diagramas UML desde el código existente.

## Conexión con el resto del ALM
Pide al alumno que documente cómo el mantenimiento continuo previene la acumulación de deuda técnica.
