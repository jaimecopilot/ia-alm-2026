# P5.1 – GENERAR TESTS UNITARIOS DESDE ESPECIFICACIONES Y CÓDIGO EXISTENTE

Actúa como un diseñador instruccional experto en formación de desarrolladores de software. Necesito que generes un ejercicio práctico detallado para un curso de IA aplicada al desarrollo de software.

## Contexto del módulo

El módulo es "IA en Testing y Aseguramiento de Calidad". Los alumnos son desarrolladores que están aprendiendo a generar tests unitarios usando IAs a partir de código fuente y especificaciones ejecutables.

## Concepto clave a transmitir

La IA puede generar tests unitarios completos a partir del código fuente o de una especificación ejecutable. La calidad de los tests depende de la precisión del prompt: especificar el framework, los tipos de tests deseados y los casos concretos a cubrir produce tests mucho más útiles que un prompt genérico. En el ALM generativo, los tests no son un añadido posterior: se generan junto con el código.

## Objetivo del ejercicio

Que los alumnos aprendan a generar tests unitarios con IA usando tres enfoques: desde código fuente, desde una especificación ejecutable y desde criterios Gherkin.

## Escenario

Proporciona una función de lógica de negocio (por ejemplo, un servicio que calcula la ocupación de salas) y una especificación ejecutable con criterios Gherkin.

## Estructura que debes generar

## 1. Introducción

Explica que los tests son parte del ALM generativo. La IA puede generarlos junto con el código.

## 2. Prompt 1: Tests desde código fuente

Proporciona la función y un prompt para generar tests unitarios con pytest. Especificar: framework, tipos de tests, casos a cubrir.

## 3. Prompt 2: Tests desde especificación ejecutable

Proporciona una especificación y un prompt para generar tests que verifiquen cada requisito.

## 4. Prompt 3: Tests desde criterios Gherkin

Proporciona criterios Gherkin para generar un test por cada escenario.

## 5. Prompt 4: Generar datos de prueba

Proporciona un prompt para generar fixtures y datos mock en formato JSON.

## 6. Tabla de evaluación

Crea una tabla con columnas para cada enfoque y filas para: cobertura de casos, legibilidad, uso de mocks, adecuación al framework.

## 7. Reflexión final

Pregunta al alumno: ¿qué enfoque produjo los tests más completos?, ¿los tests generados cubren todos los criterios Gherkin?, ¿cómo integrarías la generación de tests en tu flujo de desarrollo con IA?

## Reglas de formato

NO uses la palabra "minuto". Estructura el ejercicio en "Paso 1", "Paso 2", etc.

El ejercicio debe ocupar al alumno aproximadamente 15 minutos.

Los prompts deben estar en bloques de Markdown.

Incluye el código fuente, la especificación y espacios para los tests generados.

Los prompts deben ser genéricos y funcionar en cualquier IA.

## Tono

Profesional pero cercano.
