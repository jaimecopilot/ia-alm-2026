# P5.6 – CICLO COMPLETO: DEL REQUISITO AL CÓDIGO TESTEADO Y VALIDADO

Actúa como un diseñador instruccional experto en formación de desarrolladores de software. Necesito que generes un ejercicio práctico detallado para un curso de IA aplicada al desarrollo de software.

## Contexto del módulo
El módulo es "IA en Testing y Aseguramiento de Calidad". Este es el ejercicio integrador que cierra el módulo, combinando todas las habilidades aprendidas.

## Concepto clave a transmitir
En el ALM generativo, el testing no es una fase separada al final del desarrollo. Es una parte integrada del flujo: los tests se generan junto con el código, se ejecutan automáticamente, los fallos se corrigen en un bucle de auto-reparación, y la seguridad se valida antes del merge. El desarrollador supervisa todo el proceso.

## Objetivo del ejercicio
Que los alumnos ejecuten el ciclo completo de testing en el ALM generativo: partir de una especificación con criterios Gherkin, generar tests, ejecutarlos, reparar fallos, mejorar cobertura, validar seguridad y documentar.

## Escenario
Proporciona una especificación ejecutable con criterios Gherkin para una funcionalidad.

## Estructura que debes generar

### 1. Introducción
Explica el flujo completo del ALM generativo en testing.

### 2. Paso 1: Generar tests desde los criterios Gherkin
Proporciona un prompt para generar tests que verifiquen cada criterio.

### 3. Paso 2: Ejecutar y reparar fallos
Proporciona instrucciones para ejecutar los tests y usar la IA para reparar fallos.

### 4. Paso 3: Analizar cobertura y mejorarla
Proporciona un prompt para analizar cobertura y generar tests adicionales.

### 5. Paso 4: Validar seguridad
Proporciona un prompt para revisar el código en busca de vulnerabilidades.

### 5. Paso 5: Generar informe de calidad
Proporciona un prompt para generar un informe resumen con cobertura, tests pasando y hallazgos de seguridad.

### 6. Producto final
Indica al alumno que debe entregar: tests generados, informe de cobertura, informe de seguridad y reflexión.

### 7. Reflexión final
Pregunta al alumno: ¿cómo cambió tu percepción del testing con IA después de este ciclo completo?, ¿qué fase requirió más intervención humana?, ¿cómo integrarías este flujo en un pipeline de CI/CD?

## Reglas de formato
NO uses la palabra "minuto". Estructura el ejercicio en "Paso 1", "Paso 2", etc.

El ejercicio debe ocupar al alumno aproximadamente 15 minutos.

Los prompts deben estar en bloques de Markdown.

Incluye la especificación ejecutable y espacios para cada entregable.

Los prompts deben ser genéricos y funcionar en cualquier IA.

Tono
Profesional pero cercano. Enfatiza que este flujo es el presente del desarrollo de software, no el futuro.
