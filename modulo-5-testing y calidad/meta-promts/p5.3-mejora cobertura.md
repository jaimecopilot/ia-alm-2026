# P5.3 – MEJORA DE COBERTURA: IDENTIFICAR BRECHAS Y GENERAR TESTS

Actúa como un diseñador instruccional experto en formación de desarrolladores de software. Necesito que generes un ejercicio práctico detallado para un curso de IA aplicada al desarrollo de software.

## Contexto del módulo
El módulo es "IA en Testing y Aseguramiento de Calidad". Los alumnos están aprendiendo a usar IAs para mejorar la cobertura de tests de forma sistemática.

## Concepto clave a transmitir
La mejora de cobertura con IA es una conversación, no un comando mágico. El flujo es: ejecutar tests con cobertura, identificar líneas no cubiertas, pedir a la IA tests específicos para esas líneas, ejecutar y repetir. La IA ayuda en cada paso pero no sustituye el criterio del desarrollador sobre qué nivel de cobertura es adecuado.

## Objetivo del ejercicio
Que los alumnos aprendan a usar IAs para analizar informes de cobertura, identificar brechas y generar tests orientados a líneas o ramas no cubiertas.

## Escenario
Proporciona un módulo con tests existentes pero cobertura incompleta (algunas ramas no cubiertas, algunos métodos sin tests).

## Estructura que debes generar

### 1. Introducción
Explica que la cobertura no es solo un número: es una herramienta para encontrar puntos ciegos.

### 2. Prompt 1: Analizar informe de cobertura
Proporciona un informe de cobertura y pide a la IA que identifique las líneas no cubiertas y sugiera qué tests faltan.

### 3. Prompt 2: Generar tests para líneas no cubiertas
Proporciona las líneas específicas y pide tests que las cubran.

### 4. Prompt 3: Generar tests para ramas no cubiertas
Proporciona un informe de branch coverage y pide tests para las condiciones no evaluadas.

### 5. Prompt 4: Generar tests para edge cases
Pide a la IA que identifique edge cases no cubiertos y genere tests para ellos.

### 6. Tabla de evaluación
Crea una tabla con columnas para antes y después, y filas para: porcentaje de cobertura, número de tests, casos cubiertos.

### 7. Reflexión final
Pregunta al alumno: ¿qué tipo de brechas fueron más difíciles de cubrir?, ¿la IA generó tests redundantes o todos aportaban valor?, ¿qué porcentaje de cobertura considerarías suficiente?

## Reglas de formato
NO uses la palabra "minuto". Estructura el ejercicio en "Paso 1", "Paso 2", etc.

El ejercicio debe ocupar al alumno aproximadamente 15 minutos.

Los prompts deben estar en bloques de Markdown.

Incluye el código fuente y el informe de cobertura.

Los prompts deben ser genéricos y funcionar en cualquier IA.

Tono
Profesional pero cercano.
