# P5.5 – GENERAR TESTS DE INTEGRACIÓN Y PRUEBAS DE API

Actúa como un diseñador instruccional experto en formación de desarrolladores de software. Necesito que generes un ejercicio práctico detallado para un curso de IA aplicada al desarrollo de software.

## Contexto del módulo
El módulo es "IA en Testing y Aseguramiento de Calidad". Los alumnos están aprendiendo a generar tests de integración y pruebas de API con IAs.

## Concepto clave a transmitir
Los tests de integración verifican que los componentes funcionan correctamente cuando se conectan entre sí. En el ALM generativo, estos tests son cruciales porque validan que el código generado por la IA se integra correctamente con el resto del sistema. La IA puede generar tanto los tests como la configuración necesaria (base de datos de prueba, fixtures, clientes HTTP).

## Objetivo del ejercicio
Que los alumnos aprendan a generar tests de integración para APIs REST, incluyendo configuración de base de datos de prueba, fixtures, cliente HTTP y validación de respuestas.

## Escenario
Proporciona la definición de una API REST con varios endpoints (CRUD de salas, reservas) y pide generar tests de integración.

## Estructura que debes generar

### 1. Introducción
Explica la diferencia entre tests unitarios y de integración. En el ALM generativo, ambos son necesarios.

### 2. Prompt 1: Tests de integración para un endpoint
Proporciona un endpoint y pide a la IA que genere tests de integración con cliente HTTP, base de datos de prueba y validación de respuestas.

### 3. Prompt 2: Tests de flujo completo
Pide a la IA que genere un test que simule un flujo completo: crear sala, crear reserva, verificar disponibilidad, cancelar reserva.

### 4. Prompt 3: Tests desde OpenAPI/Swagger
Proporciona una especificación OpenAPI y pide a la IA que genere tests para todos los endpoints documentados.

### 5. Prompt 4: Generar configuración de testing
Pide a la IA que genere la configuración necesaria: base de datos en memoria, fixtures, cliente de prueba.

### 6. Tabla de evaluación
Crea una tabla con columnas para cada tipo de test y filas para: cobertura de endpoints, validación de respuestas, manejo de errores, realismo del flujo.

### 7. Reflexión final
Pregunta al alumno: ¿qué ventajas tienen los tests de integración sobre los unitarios?, ¿cómo se complementan?, ¿cómo generaría tests de integración para un sistema generado automáticamente por IA?

## Reglas de formato
NO uses la palabra "minuto". Estructura el ejercicio en "Paso 1", "Paso 2", etc.

El ejercicio debe ocupar al alumno aproximadamente 15 minutos.

Los prompts deben estar en bloques de Markdown.

Incluye la definición de la API y espacios para los tests generados.

Los prompts deben ser genéricos y funcionar en cualquier IA.

Tono
Profesional pero cercano.
