# P5.2 – DEPURACIÓN Y REPARACIÓN DE TESTS FALLIDOS CON IA

Actúa como un diseñador instruccional experto en formación de desarrolladores de software. Necesito que generes un ejercicio práctico detallado para un curso de IA aplicada al desarrollo de software.

## Contexto del módulo
El módulo es "IA en Testing y Aseguramiento de Calidad". Los alumnos están aprendiendo a usar IAs para diagnosticar y reparar tests fallidos, simulando el bucle de auto-reparación del ALM generativo.

## Concepto clave a transmitir
En el ALM generativo, cuando un agente ejecuta tests y encuentra fallos, no se detiene: diagnostica la causa raíz y corrige el código. Este bucle de auto-reparación es una de las capacidades más potentes. Los alumnos deben aprender a usar la IA para este mismo propósito: diagnosticar por qué un test falla y generar una corrección.

## Objetivo del ejercicio
Que los alumnos aprendan a usar IAs para diagnosticar tests fallidos (analizando el mensaje de error y el código), corregir el código o el test según corresponda, y verificar que la corrección funciona.

## Escenario
Proporciona varios tests fallidos con diferentes tipos de errores: mock mal configurado, aserción incorrecta, error en el código fuente, dependencia faltante.

## Estructura que debes generar

### 1. Introducción
Explica el bucle de auto-reparación: generar → testear → fallar → diagnosticar → corregir → repetir.

### 2. Prompt 1: Diagnosticar fallo por mock incorrecto
Proporciona un test que falla porque el mock no devuelve el valor esperado, junto con el mensaje de error. Pide a la IA que diagnostique y corrija.

### 3. Prompt 2: Diagnosticar fallo por aserción incorrecta
Proporciona un test que falla porque la aserción espera un valor incorrecto. Pide a la IA que determine si el error está en el test o en el código.

### 4. Prompt 3: Diagnosticar fallo por error en código fuente
Proporciona un test que falla porque el código fuente tiene un bug. Pide a la IA que diagnostique el bug y lo corrija.

### 5. Prompt 4: Reparación en lote
Proporciona varios tests fallidos y pide a la IA que diagnostique y corrija todos.

### 6. Tabla de evaluación
Crea una tabla con columnas para cada tipo de fallo y filas para: ¿diagnosticó correctamente?, ¿corrigió la causa raíz o solo el síntoma?, ¿la corrección introdujo nuevos fallos?

### 7. Reflexión final
Pregunta al alumno: ¿en qué casos la IA corrigió el test y en cuáles el código fuente?, ¿cómo se relaciona esto con el bucle de auto-reparación del ALM generativo?, ¿confiarías en correcciones automáticas sin revisión humana?

## Reglas de formato
NO uses la palabra "minuto". Estructura el ejercicio en "Paso 1", "Paso 2", etc.

El ejercicio debe ocupar al alumno aproximadamente 15 minutos.

Los prompts deben estar en bloques de Markdown.

Incluye los tests fallidos con sus mensajes de error.

Los prompts deben ser genéricos y funcionar en cualquier IA.

Tono
Profesional pero cercano.
