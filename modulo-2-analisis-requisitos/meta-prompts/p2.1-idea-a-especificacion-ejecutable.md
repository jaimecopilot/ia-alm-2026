# P2.1 – DE IDEA DE NEGOCIO A ESPECIFICACIÓN EJECUTABLE

Actúa como un diseñador instruccional experto en formación de desarrolladores de software. Necesito que generes un ejercicio práctico detallado para un curso de IA aplicada al desarrollo de software.

---

## Paso 1: Introducción

Antes de trabajar con IAs generativas, los requisitos solían redactarse en forma de historias de usuario. Sin embargo, una historia tradicional ("Como usuario quiero X para Y") rara vez contiene suficientes detalles para que una IA de codificación genere código útil automáticamente. 

Una **especificación ejecutable** va más allá: incluye contexto de negocio, detalles técnicos (endpoint, método HTTP, parámetros, framework), criterios de aceptación en formato Gherkin y restricciones explícitas. Así, cualquier IA puede transformar esa especificación directamente en código funcional. Una buena especificación es la diferencia entre obtener exactamente lo que necesitas... o un desastre.

---

## Paso 2: Elicitación conversacional con IA

Comienza transformando una idea vaga en requisitos claros gracias a la IA. Utiliza el siguiente prompt:

```markdown
Actúa como analista de negocio experto en desarrollo de software. Tengo la siguiente idea de negocio:

"Necesito un sistema para gestionar las salas de reuniones de la oficina".

Hazme preguntas una por una para clarificar y detallar esta necesidad, como si estuvieras guiando una sesión de análisis. Continúa preguntando hasta que tengas la información suficiente para redactar una especificación ejecutable.
```

## Paso 3: Generar la especificación ejecutable

Ahora utiliza el siguiente prompt, una vez tengas suficiente información obtenida en la elicitación:

```markdown
Con la información recabada, genera una especificación ejecutable. Incluye:
- Título descriptivo
- Contexto del negocio
- Propuesta de solución técnica
- Detalles técnicos concretos (endpoint, método HTTP, parámetros, tecnologías, archivos implicados)
- Criterios de aceptación en Gherkin (mínimo 4 escenarios)
- Restricciones explícitas (lo que NO debe hacer o casos que debe evitar)
La especificación debe ser clara y permitir que una IA generativa de código (Copilot, ChatGPT, Claude, etc.) pueda implementarla directamente, sin asumir nada.
```

## Paso 4: Validar la especificación ejecutable

La calidad de una especificación no depende solo de lo que incluye, sino también de lo que aclara y evita. Usa este prompt para revisar y mejorar tu resultado:

```markdown
Actúa como revisor senior de especificaciones ejecutables para agentes de IA. Revisa la siguiente especificación. Señala:
- ¿Existen ambigüedades semánticas o técnicas?
- ¿Los criterios Gherkin son comprobables y suficientemente precisos?
- ¿Los detalles técnicos bastan para que otra IA genere código funcional y correcto?
- Indica cómo mejorar la especificación si es necesario.
(Si la especificación no es suficiente, indica qué falta.)
```
