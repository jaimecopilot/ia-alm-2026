# P8.8 – PROYECTO INTEGRADOR: DOCUMENTACIÓN, DIAGRAMAS Y ADRs

## Herramientas cubiertas
Genérico (cualquier IA)

## Laboratorio generado por IA
Instrucción: El formador debe copiar este meta-prompt y pegarlo en ChatGPT, Claude o Gemini. La IA generará un laboratorio guiado completo listo para entregar al alumno.

## Actúa como un arquitecto de ALM Generativo
Estás diseñando un laboratorio práctico para un proyecto integrador donde los alumnos deben generar la documentación completa del sistema de espacios de coworking que han analizado, diseñado, implementado y testeado en los ejercicios anteriores del Módulo 8.

## Objetivo del ejercicio
Que los alumnos apliquen técnicas de generación de documentación con IA para crear docstrings, READMEs, diagramas UML actualizados desde el código real, ADRs de decisiones implementadas y un índice de trazabilidad.

## Escenario
Los alumnos ya han implementado los endpoints del sistema de coworking en P8.6 y los han testeado en P8.7. Ahora necesitan documentar todo el trabajo realizado para que el sistema sea mantenible.

## Estructura que debes generar

### 1. Introducción
Explica que la documentación en el ALM generativo no es una fase separada: es un subproducto natural que se genera desde el código implementado. La IA puede leer el código real y generar documentación actualizada.

### 2. Técnica 1: Generar docstrings para todo el código nuevo
Proporciona un prompt para que el alumno pida a la IA que genere docstrings en formato Google para todos los archivos del módulo implementado (modelos, schemas, servicios, endpoints). Incluye espacio para pegar el código documentado.

### 3. Técnica 2: Generar README del módulo
Proporciona un prompt para que la IA genere un README.md completo con: propósito del módulo, arquitectura, entidades, servicios, endpoints, configuración, ejemplos de uso y cómo ejecutar los tests.

### 4. Técnica 3: Generar diagramas desde el código real
Proporciona un prompt para que la IA lea el código implementado y genere: un diagrama de clases UML actualizado, un diagrama de secuencia del flujo principal, y un diagrama C4 de contenedores. Todos en Mermaid.

### 5. Técnica 4: Generar ADRs de decisiones implementadas
Proporciona un prompt para que la IA genere al menos 3 ADRs documentando las decisiones arquitectónicas que se tomaron durante la implementación: patrón Repository-Service, elección de base de datos, estrategia de autenticación, etc.

### 6. Técnica 5: Generar índice de trazabilidad
Proporciona un prompt para que la IA genere una matriz que conecte los criterios Gherkin de la especificación original con los tests que los verifican, los endpoints que los implementan y la documentación que los describe.

### 7. Producto final
Indica al alumno que debe entregar: código documentado, README, diagramas, ADRs y matriz de trazabilidad. Todo en formato Markdown versionable.

### 8. Reflexión final
Pregunta al alumno: ¿los diagramas generados desde código real coinciden con los diagramas de diseño iniciales? Si hay diferencias, ¿a qué se deben? ¿Cómo mantendrías esta documentación actualizada a medida que el sistema evoluciona? ¿Qué valor aporta tener la documentación como código versionable?
