# P8.9 – PROYECTO INTEGRADOR: DEVOPS Y DESPLIEGUE

## Herramientas cubiertas
Genérico (cualquier IA)

## Laboratorio generado por IA
Instrucción: El formador debe copiar este meta-prompt y pegarlo en ChatGPT, Claude o Gemini. La IA generará un laboratorio guiado completo listo para entregar al alumno.

## Actúa como un arquitecto de ALM Generativo
Estás diseñando un laboratorio práctico para un proyecto integrador donde los alumnos deben generar la infraestructura de despliegue para el sistema de coworking que han implementado, testeado y documentado.

## Objetivo del ejercicio
Que los alumnos apliquen técnicas de DevOps generativo para crear un Dockerfile optimizado, un pipeline de CI/CD, configuraciones de despliegue y un sistema de auto-reparación.

## Escenario
Los alumnos tienen el sistema de coworking completo (código, tests, documentación). Ahora necesitan empaquetarlo y desplegarlo.

## Estructura que debes generar

### 1. Introducción
Explica que en el ALM generativo, el DevOps es el eslabón que conecta el código validado con la producción. La IA puede generar todas las configuraciones necesarias.

### 2. Técnica 1: Generar Dockerfile multietapa optimizado
Proporciona un prompt para que el alumno pida a la IA que genere un Dockerfile multietapa para el proyecto, con stage de builder y runtime, usuario non-root, healthcheck y optimización de capas.

### 3. Técnica 2: Generar pipeline CI/CD
Proporciona un prompt para que el alumno pida un pipeline de GitHub Actions o GitLab CI que incluya: lint, test con cobertura, build de imagen Docker, push al registro y deploy a staging.

### 4. Técnica 3: Generar docker-compose para desarrollo
Proporciona un prompt para generar un docker-compose.yml con la API, PostgreSQL y Redis, con volúmenes persistentes y healthchecks.

### 5. Técnica 4: Generar manifiestos Kubernetes
Proporciona un prompt para generar deployment, service y configmap para Kubernetes, con resource limits, probes y securityContext.

### 6. Técnica 5: Configurar Self-Healing CI
Proporciona un prompt para que el alumno pida a la IA que diseñe un workflow de auto-reparación: cuando el pipeline falle en main, se crea un issue automático asignado a un agente de IA.

### 7. Producto final
Indica al alumno que debe entregar: Dockerfile, docker-compose.yml, pipeline CI/CD, manifiestos Kubernetes y diseño del sistema Self-Healing CI.

### 8. Reflexión final
Pregunta al alumno: ¿qué ventajas tiene generar configuraciones de infraestructura con IA frente a escribirlas manualmente? ¿Qué riesgos hay que considerar? ¿Cómo validarías las configuraciones generadas antes de aplicarlas en producción? ¿Cómo se integra el Self-Healing CI en el flujo completo del ALM generativo?
