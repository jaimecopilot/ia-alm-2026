# P8.7 – PROYECTO INTEGRADOR: TESTING, REVISIÓN DE SEGURIDAD Y CI/CD

Actúa como un diseñador instruccional experto en formación de desarrolladores de software. Necesito que generes un ejercicio práctico detallado para un curso de IA aplicada al desarrollo de software.

## Contexto
Tercer ejercicio del proyecto integrador. Los alumnos aplicarán testing, seguridad y CI/CD al código implementado.

## Concepto clave a transmitir
En el ALM generativo, el testing, la seguridad y el CI/CD no son fases separadas sino integradas: los tests se generan con el código, la seguridad se valida automáticamente y el pipeline se configura con IA.

## Objetivo del ejercicio
Que los alumnos generen tests de integración, realicen una revisión de seguridad con IA, y configuren un pipeline de CI/CD.

## Estructura que debes generar

### 1. Introducción
Explica la integración de testing, seguridad y CI/CD en el ALM generativo.

### 2. Fase 6: Generar tests de integración
Pide al alumno que genere tests de integración para los endpoints implementados.

### 3. Fase 7: Revisión de seguridad
Pide al alumno que use IAs para revisar el código en busca de vulnerabilidades.

### 4. Fase 8: Configurar CI/CD
Pide al alumno que use IAs para generar un pipeline de CI/CD.

### 5. Entregable
Tests de integración, informe de seguridad y pipeline CI/CD.

### 6. Reflexión parcial
Pregunta al alumno: ¿la IA detectó vulnerabilidades reales?, ¿el pipeline generado era funcional?

## Regles de formato
NO uses la palabra "minuto". Estructura el ejercicio en "Paso 1", "Paso 2", etc.

El ejercicio debe ocupar al alumno aproximadamente 15 minutos.

Los prompts deben ser genéricos y funcionar en cualquier IA.

Incluye espacios para los entregables.

## Tono
Profesional pero cercano.
