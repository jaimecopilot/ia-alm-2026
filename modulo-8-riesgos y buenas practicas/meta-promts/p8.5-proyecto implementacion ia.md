# P8.6 – PROYECTO INTEGRADOR: IMPLEMENTACIÓN CON IA

Actúa como un diseñador instruccional experto en formación de desarrolladores de software. Necesito que generes un ejercicio práctico detallado para un curso de IA aplicada al desarrollo de software.

## Contexto
Segundo ejercicio del proyecto integrador. Los alumnos implementarán el código a partir de la especificación y el diseño generados en P8.5.

## Concepto clave a transmitir
La implementación en el ALM generativo consiste en transformar especificaciones ejecutables en código funcional multi-archivo mediante IAs, aplicando patrones de diseño y buenas prácticas.

## Objetivo del ejercicio
Que los alumnos generen el código de al menos dos endpoints del sistema usando IAs, aplicando patrones Repository-Service y generando tests unitarios.

## Estructura que debes generar

### 1. Introducción
Explica que la implementación es la fase donde los requisitos se convierten en código.

### 2. Fase 3: Generar código desde la especificación
Pide al alumno que use la especificación ejecutable como prompt para generar el código.

### 3. Fase 4: Refactorizar y aplicar patrones
Pide al alumno que refactorice el código generado aplicando Clean Code y patrones de diseño.

### 4. Fase 5: Generar tests unitarios
Pide al alumno que genere tests unitarios para el código implementado.

### 5. Entregable
Código fuente, tests unitarios y breve informe de decisiones técnicas.

### 6. Reflexión parcial
Pregunta al alumno: ¿la IA generó código correcto a la primera?, ¿cuántas iteraciones fueron necesarias?, ¿qué IAs usaste?

## Regles de formato
NO uses la palabra "minuto". Estructura el ejercicio en "Paso 1", "Paso 2", etc.

El ejercicio debe ocupar al alumno aproximadamente 15 minutos.

Los prompts deben ser genéricos y funcionar en cualquier IA.

Incluye espacios para los entregables.

## Tono
Profesional pero cercano.
