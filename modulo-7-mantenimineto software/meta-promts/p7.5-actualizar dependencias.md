# P7.5 – ACTUALIZAR DEPENDENCIAS Y DETECTAR VULNERABILIDADES

Actúa como un diseñador instruccional experto en formación de desarrolladores de software. Necesito que generes un ejercicio práctico detallado para un curso de IA aplicada al desarrollo de software.

## Contexto del módulo

El módulo es "IA en Mantenimiento y Evolución del Software". Los alumnos están aprendiendo a usar IAs para gestionar dependencias y detectar vulnerabilidades.

## Concepto clave a transmitir

La gestión de dependencias es una tarea crítica de mantenimiento. La IA puede analizar archivos de dependencias, identificar versiones desactualizadas, detectar vulnerabilidades conocidas y planificar actualizaciones seguras priorizando por riesgo.

## Objetivo del ejercicio

Que los alumnos aprendan a usar IAs para auditar dependencias, detectar vulnerabilidades, planificar actualizaciones y verificar compatibilidad.

## Escenario

Proporciona un archivo de dependencias (requirements.txt o package.json) con versiones desactualizadas y alguna dependencia con vulnerabilidad conocida.

## Estructura que debes generar

### 1. Introducción

Explica la importancia de mantener las dependencias actualizadas y cómo la IA puede ayudar.

### 2. Prompt 1: Auditar dependencias

Proporciona el archivo y pide a la IA que identifique versiones desactualizadas y sin especificar.

### 3. Prompt 2: Detectar vulnerabilidades

Proporciona un prompt para que la IA identifique dependencias con vulnerabilidades conocidas.

### 4. Prompt 3: Planificar actualizaciones

Proporciona un prompt para generar un plan de actualización priorizado (seguridad primero, luego menores, luego mayores).

### 5. Prompt 4: Verificar compatibilidad

Proporciona un prompt para que la IA analice posibles breaking changes entre versiones.

### 6. Tabla de evaluación

Crea una tabla con las dependencias y su estado: actual, recomendada, riesgo, breaking changes.

### 7. Reflexión final

Pregunta al alumno: ¿la IA identificó correctamente las vulnerabilidades?, ¿el plan de actualización era realista?, ¿cómo automatizarías este proceso en CI/CD?

## Reglas de formato

NO uses la palabra "minuto". Estructura el ejercicio en "Paso 1", "Paso 2", etc.

El ejercicio debe ocupar al alumno aproximadamente 15 minutos.

Los prompts deben estar en bloques de código Markdown.

Incluye el archivo de dependencias y espacios para el análisis.

Los prompts deben ser genéricos y funcionar en cualquier IA.

## Tono

Profesional pero cercano.
