# P7.4 – PLANIFICAR Y EJECUTAR REFACTORIZACIONES CON IA

Actúa como un diseñador instruccional experto en formación de desarrolladores de software. Necesito que generes un ejercicio práctico detallado para un curso de IA aplicada al desarrollo de software.

## Contexto del módulo
El módulo es "IA en Mantenimiento y Evolución del Software". Los alumnos están aprendiendo a planificar y ejecutar refactorizaciones usando IAs.

## Concepto clave a transmitir
La refactorización con IA no es solo pedir "mejora este código". Consiste en describir exactamente qué principios aplicar, proporcionar el código original y las restricciones, y verificar que el comportamiento no cambia mediante tests de caracterización. En el ALM generativo, los agentes pueden ejecutar refactorizaciones multi-archivo de forma autónoma.

## Objetivo del ejercicio
Que los alumnos aprendan a planificar una refactorización basada en deuda técnica detectada, ejecutarla con IA y verificar que el comportamiento no cambia.

## Escenario
Proporciona el mismo código con deuda técnica del ejercicio anterior y el backlog de refactorización generado.

## Estructura que debes generar

### 1. Introducción
Explica que la refactorización segura requiere planificación, ejecución y verificación.

### 2. Prompt 1: Refactorizar nombres y type hints
Proporciona un prompt para aplicar la primera refactorización del backlog.

### 3. Prompt 2: Extraer métodos y reducir complejidad
Proporciona un prompt para dividir métodos largos y reducir anidamiento.

### 4. Prompt 3: Eliminar duplicación
Proporciona un prompt para identificar y extraer código duplicado.

### 5. Prompt 4: Verificar con tests de caracterización
Proporciona un prompt para ejecutar los tests generados en P7.1 y verificar que todo sigue pasando.

### 6. Comparación antes/después
Incluye espacio para comparar el código original con el refactorizado.

### 7. Reflexión final
Pregunta al alumno: ¿la refactorización mejoró la legibilidad?, ¿los tests de caracterización detectaron algún cambio de comportamiento?, ¿cómo abordarías refactorizaciones en un proyecto real sin tests?

## Reglas de formato
NO uses la palabra "minuto". Estructura el ejercicio en "Paso 1", "Paso 2", etc.

El ejercicio debe ocupar al alumno aproximadamente 15 minutos.

Los prompts deben estar en bloques de código Markdown.

Incluye el código original y espacios para el código refactorizado.

Los prompts deben ser genéricos y funcionar en cualquier IA.

## Tono
Profesional pero cercano.
