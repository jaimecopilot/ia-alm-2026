# P7.6 – MODERNIZAR CÓDIGO: MIGRACIÓN DE FRAMEWORKS Y LENGUAJES

Actúa como un diseñador instruccional experto en formación de desarrolladores de software. Necesito que generes un ejercicio práctico detallado para un curso de IA aplicada al desarrollo de software.

## Contexto del módulo

El módulo es "IA en Mantenimiento y Evolución del Software". Los alumnos están aprendiendo a usar IAs para modernizar código, migrando entre frameworks, lenguajes y estilos.

## Concepto clave a transmitir

La modernización de código con IA no es solo traducción sintáctica: implica adaptar convenciones idiomáticas, aprovechar las características del nuevo lenguaje o framework, y verificar que el comportamiento no cambia mediante tests de caracterización. La IA acelera la parte mecánica; el desarrollador valida el resultado.

## Objetivo del ejercicio

Que los alumnos aprendan a migrar código entre frameworks (Flask a FastAPI), entre lenguajes (Java a Python) y entre estilos (ES5 a TypeScript), verificando la equivalencia funcional.

## Escenario

Proporciona fragmentos de código en diferentes lenguajes/frameworks para migrar.

## Estructura que debes generar

### 1. Introducción

Explica que la modernización es una de las tareas más arriesgadas y cómo la IA puede ayudar.

### 2. Prompt 1: Migrar Flask a FastAPI

Proporciona una API Flask y pide migrarla a FastAPI manteniendo la misma lógica y códigos de respuesta.

### 3. Prompt 2: Migrar Java a Python

Proporciona una clase Java y pide traducirla a Python adaptando convenciones idiomáticas.

### 4. Prompt 3: Migrar ES5 a TypeScript

Proporciona código JavaScript ES5 y pide migrarlo a TypeScript con tipos estrictos.

### 5. Prompt 4: Verificar equivalencia

Proporciona un prompt para generar tests que verifiquen que el código migrado produce los mismos resultados.

### 6. Tabla de evaluación

Crea una tabla con columnas para cada migración y filas para: fidelidad de la lógica, uso de convenciones idiomáticas, legibilidad, cobertura de tests.

### 7. Reflexión final

Pregunta al alumno: ¿qué migración fue más compleja?, ¿la IA respetó las convenciones del lenguaje destino?, ¿cómo verificarías que el código migrado es correcto antes de ponerlo en producción?

## Reglas de formato

NO uses la palabra "minuto". Estructura el ejercicio en "Paso 1", "Paso 2", etc.

El ejercicio debe ocupar al alumno aproximadamente 15 minutos.

Los prompts deben estar en bloques de código Markdown.

Incluye el código original y espacios para el código migrado.

Los prompts deben ser genéricos y funcionar en cualquier IA.

## Tono

Profesional pero cercano.
