# P7.3 – DETECTAR DEUDA TÉCNICA Y CODE SMELLS CON IA

Actúa como un diseñador instruccional experto en formación de desarrolladores de software. Necesito que generes un ejercicio práctico detallado para un curso de IA aplicada al desarrollo de software.

Contexto del módulo  
El módulo es "IA en Mantenimiento y Evolución del Software". Los alumnos están aprendiendo a usar IAs para detectar deuda técnica y code smells.

Concepto clave a transmitir  
La IA puede actuar como un auditor automático de código, identificando code smells, duplicación, complejidad excesiva y otros indicadores de deuda técnica. Puede priorizar los hallazgos y sugerir refactorizaciones concretas. En el ALM generativo, esta auditoría puede ejecutarse automáticamente en cada PR, previniendo la acumulación de nueva deuda técnica.

Objetivo del ejercicio  
Que los alumnos aprendan a usar IAs para analizar código en busca de deuda técnica, cuantificarla y generar un backlog de refactorización priorizado.

Escenario  
Proporciona un módulo de código con deuda técnica intencionada: métodos largos, duplicación, nombres poco descriptivos, falta de type hints, alta complejidad ciclomática, imports no utilizados.

Estructura que debes generar  
1. Introducción  
Explica qué es la deuda técnica y cómo la IA puede ayudar a detectarla.

2. Prompt 1: Identificar code smells  
Proporciona el código y pide a la IA que enumere todos los code smells, clasificándolos por tipo y severidad.

3. Prompt 2: Cuantificar deuda técnica  
Proporciona un prompt para que la IA estime el tiempo necesario para resolver cada smell.

4. Prompt 3: Priorizar refactorizaciones  
Proporciona un prompt para priorizar los hallazgos (quick wins primero, cambios mayores después).

5. Prompt 4: Generar backlog de refactorización  
Proporciona un prompt para generar un backlog en formato tabla con: smell, archivo, prioridad, esfuerzo estimado y sugerencia de refactorización.

6. Tabla de evaluación  
Crea una tabla con los smells identificados y pide al alumno que evalúe si la IA acertó en el diagnóstico.

7. Reflexión final  
Pregunta al alumno: ¿la IA identificó todos los code smells?, ¿las prioridades asignadas eran correctas?, ¿cómo integrarías esta auditoría en el flujo de desarrollo?

Reglas de formato  
NO uses la palabra "minuto". Estructura el ejercicio en "Paso 1", "Paso 2", etc.

El ejercicio debe ocupar al alumno aproximadamente 15 minutos.

Los prompts deben estar en bloques de código Markdown.

Incluye el código con deuda técnica y espacios para el análisis.

Los prompts deben ser genéricos y funcionar en cualquier IA.

Tono  
Profesional pero cercano.
