# P7.2 – GENERAR DOCUMENTACIÓN AUTOMÁTICA: DOCSTRINGS, README Y ADRS

Actúa como un diseñador instruccional experto en formación de desarrolladores de software. Necesito que generes un ejercicio práctico detallado para un curso de IA aplicada al desarrollo de software.

Contexto del módulo  
El módulo es "IA en Mantenimiento y Evolución del Software". Los alumnos están aprendiendo a generar documentación automática desde código fuente usando IAs.

Concepto clave a transmitir  
La documentación no tiene por qué escribirse manualmente. La IA puede generar docstrings, READMEs, diagramas de arquitectura y ADRs directamente desde el código fuente. Esta documentación es "viva": puede regenerarse cuando el código cambia, asegurando que siempre refleja la implementación real. En el ALM generativo, la documentación es un subproducto natural del desarrollo, no una tarea separada.

Objetivo del ejercicio  
Que los alumnos aprendan a generar diferentes tipos de documentación desde código fuente: docstrings, README de módulo, diagramas C4, ADRs retrospectivos e índice de documentación.

Escenario  
Proporciona un módulo de código sin documentar (puede ser el mismo del ejercicio anterior u otro).

Estructura que debes generar  
1. Introducción  
Explica el concepto de documentación viva generada por IA.

2. Prompt 1: Generar docstrings  
Proporciona el código y un prompt para generar docstrings en formato Google para todas las funciones y clases.

3. Prompt 2: Generar README del módulo  
Proporciona un prompt para generar un README con propósito, estructura, dependencias y ejemplos.

4. Prompt 3: Generar diagramas C4  
Proporciona un prompt para generar diagramas C4 de contexto y contenedores en Mermaid basados en el código.

5. Prompt 4: Generar ADRs retrospectivos  
Proporciona un prompt para que la IA infiera decisiones arquitectónicas del código y genere ADRs.

6. Prompt 5: Generar índice de documentación  
Proporciona un prompt para generar un índice que unifique todos los artefactos de documentación.

7. Tabla de evaluación  
Crea una tabla con columnas para cada artefacto y filas para: ¿refleja correctamente el código?, ¿es útil para nuevos desarrolladores?, ¿está bien estructurado?

8. Reflexión final  
Pregunta al alumno: ¿la documentación generada es directamente utilizable?, ¿qué partes requirieron más refinamiento?, ¿cómo mantenerías esta documentación actualizada a medida que el código evoluciona?

Reglas de formato  
NO uses la palabra "minuto". Estructura el ejercicio en "Paso 1", "Paso 2", etc.

El ejercicio debe ocupar al alumno aproximadamente 15 minutos.

Los prompts deben estar en bloques de código Markdown.

Incluye el código fuente y espacios para la documentación generada.

Los prompts deben ser genéricos y funcionar en cualquier IA.

Tono  
Profesional pero cercano.
