# P7.1 – EXPLICAR CÓDIGO LEGADO Y GENERAR DIAGRAMAS DESDE ÉL

Actúa como un diseñador instruccional experto en formación de desarrolladores de software. Necesito que generes un ejercicio práctico detallado para un curso de IA aplicada al desarrollo de software.

Contexto del módulo  
El módulo es "IA en Mantenimiento y Evolución del Software". Los alumnos son desarrolladores que están aprendiendo a usar IAs para entender código legacy y generar documentación visual a partir de él.

Concepto clave a transmitir  
La IA es una herramienta de primera línea para abordar código desconocido. Puede explicar su funcionamiento a diferentes niveles de profundidad, identificar dependencias ocultas, señalar código muerto y generar diagramas de su estructura. Esto acelera drásticamente la incorporación a proyectos heredados y forma parte del mantenimiento continuo en el ALM generativo.

Objetivo del ejercicio  
Que los alumnos aprendan a usar IAs para entender código legacy complejo, obtener explicaciones a diferentes niveles, identificar problemas y generar diagramas UML y de flujo.

Escenario  
Proporciona un fragmento de código legacy intencionadamente complejo: sin documentación, con nombres crípticos, estructuras anidadas y lógica de negocio no evidente. Puede ser en un lenguaje moderno pero con malas prácticas, o en un lenguaje antiguo.

Estructura que debes generar  
1. Introducción  
Explica que el análisis de código legacy es una de las aplicaciones más valiosas de la IA en mantenimiento.

2. Prompt 1: Explicación general  
Proporciona el código y un prompt para obtener una visión general de su propósito y funcionamiento.

3. Prompt 2: Explicación línea por línea  
Proporciona un prompt para obtener una explicación detallada línea por línea, incluyendo qué hace cada parte y por qué.

4. Prompt 3: Identificar problemas  
Proporciona un prompt para que la IA identifique bugs potenciales, edge cases no manejados, código muerto y dependencias ocultas.

5. Prompt 4: Generar diagramas  
Proporciona un prompt para generar un diagrama de clases UML y un diagrama de flujo en Mermaid a partir del código.

6. Prompt 5: Generar tests de caracterización  
Proporciona un prompt para generar tests que capturen el comportamiento actual del código.

7. Tabla de evaluación  
Crea una tabla con columnas para cada tipo de análisis y filas para: claridad, precisión, utilidad práctica.

8. Reflexión final  
Pregunta al alumno: ¿la IA entendió correctamente la lógica de negocio?, ¿identificó todos los problemas?, ¿cómo usarías esta capacidad al incorporarte a un proyecto con código legacy?

Reglas de formato  
NO uses la palabra "minuto". Estructura el ejercicio en "Paso 1", "Paso 2", etc.

El ejercicio debe ocupar al alumno aproximadamente 15 minutos.

Los prompts deben estar en bloques de código Markdown.

Incluye el código legacy y espacios para las explicaciones y diagramas.

Los prompts deben ser genéricos y funcionar en cualquier IA.

Tono  
Profesional pero cercano.
