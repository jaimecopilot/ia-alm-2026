# P6.6 – CICLO COMPLETO: DEL COMMIT AL DESPLIEGUE AUTOMATIZADO

Actúa como un diseñador instruccional experto en formación de desarrolladores de software. Necesito que generes un ejercicio práctico detallado para un curso de IA aplicada al desarrollo de software.

## Contexto del módulo

El módulo es "IA en DevOps y Despliegue". Este es el ejercicio integrador que cierra el módulo.

## Concepto clave a transmitir

En el ALM generativo, el DevOps es un flujo continuo: código → tests → build → deploy → monitorizar → auto-reparar. La IA asiste en cada fase. El desarrollador configura las reglas y supervisa el proceso, pero las tareas mecánicas están automatizadas.

## Objetivo del ejercicio

Que los alumnos ejecuten el ciclo completo de DevOps con IA: generar un pipeline, generar infraestructura, diagnosticar un fallo simulado, diseñar el flujo de auto-reparación y documentar.

## Escenario

Proporciona el contexto de un proyecto completo que necesita CI/CD, contenedores y despliegue cloud.

## Estructura que debes generar

### 1. Introducción

Explica el flujo completo del ALM generativo en DevOps.

### 2. Paso 1: Generar pipeline CI/CD

Proporciona un prompt para generar el pipeline completo.

### 3. Paso 2: Generar Dockerfile y manifiestos

Proporciona un prompt para generar la configuración de contenedores.

### 4. Paso 3: Generar infraestructura cloud

Proporciona un prompt para generar la infraestructura de despliegue.

### 5. Paso 4: Simular fallo y diagnosticar

Proporciona un log de fallo y pide diagnóstico y corrección.

### 6. Paso 5: Diseñar flujo de auto-reparación

Proporciona un prompt para diseñar el sistema Self-Healing CI.

### 7. Producto final

Indica al alumno que debe entregar: pipeline, Dockerfile, infraestructura, diagnóstico y diseño de auto-reparación.

### 8. Reflexión final

Pregunta al alumno: ¿cómo cambió tu percepción del DevOps con IA?, ¿qué fase fue más automatizable?, ¿qué fase requirió más intervención humana?

## Reglas de formato

- NO uses la palabra "minuto". Estructura el ejercicio en "Paso 1", "Paso 2", etc.
- El ejercicio debe ocupar al alumno aproximadamente 15 minutos.
- Los prompts deben estar en bloques de código Markdown.
- Incluye espacios para cada entregable.
- Los prompts deben ser genéricos y funcionar en cualquier IA.

## Tono

Profesional pero cercano. Enfatiza que este flujo es el presente del desarrollo de software, no el futuro.
