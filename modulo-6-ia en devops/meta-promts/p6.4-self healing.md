# P6.4 – CONFIGURAR SELF-HEALING CI CON IA

Actúa como un diseñador instruccional experto en formación de desarrolladores de software. Necesito que generes un ejercicio práctico detallado para un curso de IA aplicada al desarrollo de software.

## Contexto del módulo

El módulo es "IA en DevOps y Despliegue". Los alumnos están aprendiendo el concepto de Self-Healing CI y cómo implementarlo.

## Concepto clave a transmitir

El Self-Healing CI es la culminación del ALM generativo en DevOps: cuando un pipeline falla, un agente de IA diagnostica automáticamente, genera una corrección y abre una PR. El sistema incluye protecciones para evitar bucles infinitos. El desarrollador solo interviene cuando el agente no puede resolver el problema.

## Objetivo del ejercicio

Que los alumnos entiendan el flujo completo de Self-Healing CI y aprendan a diseñar un sistema que detecte fallos automáticamente, cree issues, asigne a un agente de IA y obtenga PRs de corrección.

## Escenario

Proporciona el contexto de un proyecto con CI/CD configurado. El alumno debe diseñar el flujo de Self-Healing CI.

## Estructura que debes generar

### 1. Introducción

Explica el concepto de Self-Healing CI y su lugar en el ALM generativo.

### 2. Prompt 1: Diseñar el flujo de Self-Healing CI

Proporciona un prompt para que la IA genere un diagrama y una descripción del flujo completo.

### 3. Prompt 2: Generar el workflow de detección

Proporciona un prompt para generar un workflow que detecte fallos y cree issues automáticamente.

### 4. Prompt 3: Definir protecciones

Proporciona un prompt para que la IA sugiera protecciones: deduplicación, límite de reintentos, prevención de bucles.

### 5. Prompt 4: Simular un ciclo completo

Proporciona un prompt para simular un fallo y la respuesta del sistema paso a paso.

### 6. Tabla de evaluación

Crea una tabla con columnas para cada componente del sistema y filas para: ¿está correctamente diseñado?, ¿incluye protecciones?, ¿es escalable?

### 7. Reflexión final

Pregunta al alumno: ¿qué riesgos tiene un sistema Self-Healing CI sin supervisión?, ¿cómo equilibrar automatización y control humano?, ¿en qué tipos de proyectos implementarías esto?

## Reglas de formato

- NO uses la palabra "minuto". Estructura el ejercicio en "Paso 1", "Paso 2", etc.
- El ejercicio debe ocupar al alumno aproximadamente 15 minutos.
- Los prompts deben estar en bloques de código Markdown.
- Incluye espacios para diagramas y configuraciones.
- Los prompts deben ser genéricos y funcionar en cualquier IA.

## Tono

Profesional pero cercano.
