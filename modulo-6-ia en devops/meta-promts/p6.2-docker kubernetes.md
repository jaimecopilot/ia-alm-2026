# P6.2 – GENERAR DOCKERFILES Y MANIFIESTOS KUBERNETES

Actúa como un diseñador instruccional experto en formación de desarrolladores de software. Necesito que generes un ejercicio práctico detallado para un curso de IA aplicada al desarrollo de software.

## Contexto del módulo

El módulo es "IA en DevOps y Despliegue". Los alumnos están aprendiendo a generar configuraciones de contenedores y orquestación con IAs.

## Concepto clave a transmitir

La IA conoce las mejores prácticas de Docker y Kubernetes y puede aplicarlas automáticamente al generar configuraciones. No necesitas conocer la sintaxis exacta: describes lo que necesitas y la IA genera archivos profesionales con health checks, resource limits, security context y demás buenas prácticas.

## Objetivo del ejercicio

Que los alumnos aprendan a generar Dockerfiles multietapa optimizados, archivos docker-compose para desarrollo local y manifiestos Kubernetes para producción.

## Escenario

Proporciona el contexto de una aplicación Python con FastAPI, PostgreSQL y Redis.

## Estructura que debes generar

### 1. Introducción

Explica que la infraestructura como código también puede generarse con IA.

### 2. Prompt 1: Dockerfile multietapa optimizado

Proporciona un prompt para generar un Dockerfile con stage de builder y runtime, usuario non-root y healthcheck.

### 3. Prompt 2: Docker-compose para desarrollo

Proporciona un prompt para generar un docker-compose con API, PostgreSQL y Redis, con volúmenes y redes.

### 4. Prompt 3: Manifiestos Kubernetes

Proporciona un prompt para generar deployment, service, configmap y hpa para la aplicación.

### 5. Prompt 4: Validar configuraciones generadas

Proporciona un prompt para que la IA revise las configuraciones en busca de problemas de seguridad o malas prácticas.

### 6. Tabla de evaluación

Crea una tabla con columnas para cada artefacto y filas para: buenas prácticas aplicadas, seguridad, optimización, completitud.

### 7. Reflexión final

Pregunta al alumno: ¿qué buenas prácticas aplicó la IA automáticamente?, ¿qué configuraciones requirieron más refinamiento?, ¿cómo verificarías que las configuraciones generadas funcionan correctamente?

## Reglas de formato

- NO uses la palabra "minuto". Estructura el ejercicio en "Paso 1", "Paso 2", etc.
- El ejercicio debe ocupar al alumno aproximadamente 15 minutos.
- Los prompts deben estar en bloques de código Markdown.
- Incluye espacios para pegar las configuraciones generadas.
- Los prompts deben ser genéricos y funcionar en cualquier IA.

## Tono

Profesional pero cercano.
