# P6.5 – GENERAR INFRAESTRUCTURA COMO CÓDIGO (TERRAFORM, BICEP)

Actúa como un diseñador instruccional experto en formación de desarrolladores de software. Necesito que generes un ejercicio práctico detallado para un curso de IA aplicada al desarrollo de software.

## Contexto del módulo

El módulo es "IA en DevOps y Despliegue". Los alumnos están aprendiendo a generar infraestructura como código usando IAs.

## Concepto clave a transmitir

La IA conoce las mejores prácticas de las principales plataformas cloud (AWS, Azure, GCP) y puede generar configuraciones Terraform o Bicep completas desde descripciones en lenguaje natural. La infraestructura se describe, no se programa.

## Objetivo del ejercicio

Que los alumnos aprendan a describir infraestructura cloud en lenguaje natural y obtener configuraciones Terraform y Bicep completas y seguras.

## Escenario

Proporciona el contexto de una aplicación web que necesita: balanceador de carga, 2 instancias de aplicación, base de datos gestionada, caché Redis y red privada.

## Estructura que debes generar

### 1. Introducción

Explica que la infraestructura como código también puede generarse con IA.

### 2. Prompt 1: Infraestructura en Azure con Terraform

Proporciona un prompt para generar configuración Terraform para Azure.

### 3. Prompt 2: Infraestructura en Azure con Bicep

Proporciona un prompt para generar lo mismo en Bicep. Comparar ambos.

### 4. Prompt 3: Infraestructura en AWS con Terraform

Proporciona un prompt para generar configuración para AWS.

### 5. Prompt 4: Validar configuraciones generadas

Proporciona un prompt para que la IA revise seguridad, costes y mejores prácticas.

### 6. Tabla de evaluación

Crea una tabla con columnas para cada plataforma/herramienta y filas para: recursos generados, seguridad, buenas prácticas, completitud.

### 7. Reflexión final

Pregunta al alumno: ¿qué diferencias hay entre generar infraestructura para distintas clouds?, ¿la IA aplicó buenas prácticas de seguridad automáticamente?, ¿cómo validarías la infraestructura generada antes de aplicarla?

## Reglas de formato

- NO uses la palabra "minuto". Estructura el ejercicio en "Paso 1", "Paso 2", etc.
- El ejercicio debe ocupar al alumno aproximadamente 15 minutos.
- Los prompts deben estar en bloques de código Markdown.
- Incluye espacios para pegar las configuraciones generadas.
- Los prompts deben ser genéricos y funcionar en cualquier IA.

## Tono

Profesional pero cercano.
